## 5. v96 で確立した永続教訓

### A. CSS token 不在の silent fail
shimmer で `--bg-surface-2/3` が未定義 → gradient が transparent で「静止」 に見える (animation は動作)。 user は「壊れている」 と認識。 教訓: 新規 token を CSS で使う前に `:root` 定義確認、 design-system-check skill で grep verify。

### B. Sprint 連続追加で endpoint regression
Sprint H6 + H8 で profile-summary に schema + 入力追加、 H7+H8 deploy 後に 503 発生。 教訓: LLM endpoint への structural 変更 (schema / few-shot / message format) は **個別 commit + smoke test** で 1 つずつ verify、 batch deploy 禁止。

### C. inline style override は CSS specificity (1,0,0) で外部 CSS を負かす
Sprint G1 で SummaryBrief.jsx の `<section style={{ border }}>` が `.verdict-hero .summary-brief-section { border: none }` を override、 user 「継ぎ目」 と認識。 `!important` + `[style]` attribute selector で打ち消した教訓。
