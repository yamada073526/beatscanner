### 2-1. 🔴 (検証) TWR デプロイ後の本番動作確認 + ユーザー納得確認

**確認項目** (本番で):
- ホーム画面のポートフォリオ推移チャートが `+3195%` から「現実的な数十% レンジ」に変わっているか
- SPY 比較バッジ (`vs SPY +X%`) が TWR ベースで再計算されているか
- ヘッダー delta が `$ 表示` から `% 投資成果` 表示に変わっているか
- pd-row hover で **cyan border + 外側 shadow + 1px 浮き上がり** すべて発火するか

**もし TWR が依然として桁外れの値**を出す場合の原因仮説 (金融 agent 指摘):
1. `lot.trade_date` がユーザー入力ミスで「過去保有を今日入金扱い」になっている → cashflow が過大
2. backend `cashflow` フィールドが正しくシリアライズされていない (旧 build キャッシュ等)
3. anchor 日 (`startIdx`) が想定外の位置で 100 起点

**初動デバッグ**: ブラウザ DevTools Network タブで `/api/portfolio-history` のレスポンスを確認、`cashflow` 値が含まれているかチェック。

### 2-2. 🔴 portfolio-history endpoint のセキュリティ穴 (Web 開発 agent 指摘)

**現状**: `POST /api/portfolio-history` は `lots` を frontend から受け取りそのまま処理。**認証なし・user_id 検証なし**。他人の任意 lots を投げて任意 ticker の履歴を引け、レート制限もないため DoS 経路。

**修正**:
- `Authorization: Bearer <jwt>` 必須化 (`_verify_supabase_jwt` 流用)
- lots は frontend 渡しでなく **server で `holding_lots` を user_id で SELECT** に変更
- IP rate limit 10 req/min
- tickers 上限 20 個

**工数**: 半日 / **優先度**: 🔴 (公開後の事故防止)

## 3. 🟡 重要知見 — 永続化必須 (v48 セッション固有)

### 3-1. TWR 計算の「初日処理」に注意

`computeTWR` の anchor 選定:
- 「value > 0 か cashflow > 0 が初めて出る日」を anchor とし、それ以降のみ TWR を構築
- **anchor 日は TWR=100 の起点**、その前の `value=0` 期間は出力に含めない
- これにより「保有してない期間の崖落ち」を回避

### 3-2. cashflow を frontend で再構成しない

frontend には trade_date 当日の close 価格情報がない。**backend が `series[i].cashflow` を返す設計が必須** (金融 agent 強調)。
- `backend/main.py:1138-` で `if l["trade_date"] == d_iso: cashflow += l["shares"] * close` を追加済み
- 将来「売却 (lot 削除)」を実装する時は cashflow を負値で扱う

### 3-3. pd-row 浮き上がり: transform + z-index:2 の組合せ

- `transform: translateY(-1px)` を使うと隣行と重なるリスク → `z-index: 2` で重ね順を最上位に固定
- Linear 流の `cubic-bezier(0.16, 1, 0.3, 1)` を採用 (UI/UX agent 推奨)
- inset 3px (cyan accent) + 外側 box-shadow を **1 つの宣言で記述** (複数 shadow を カンマ区切り)
- `.pd-row` に `position: relative; z-index: 0;` をデフォルト付与しないと z-index:2 が効かない

### 3-4. priceFormat custom は v5 で OK だが変更されると chart 全体再構築

`useEffect` deps に `series, twrSeries, status, spyPoints, showSpy` を含める。`twrSeries` 追加で再構築頻度が増えるが、`useMemo` で依存安定化済み。
