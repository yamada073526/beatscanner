## 1. v107 hotfix 詳細 (React #310 真っ白事故)

**真因**: v105 ch2Tab + v106 ch3Tab の `useState` 2 件が `JudgmentDetail.jsx` の `if (!selectedTicker) return (...)` early return より **後ろ** に配置されていた → 初回 render (selectedTicker=undefined) は hooks 2 個 skip / 2 回目 (set 後) は 2 個追加 → React error #310 "Rendered more hooks than during the previous render"。 workspace mode の全 user で真っ白事故。

**fix**: useState 2 件 + setter wrapper を `analyzedTickerRef` (L321) の直後、 early return (L338) の前に移動。 Rules of Hooks 遵守。

**復旧確認**: production `index-CVvQ_pX7.js` で DOM child count 0→1、 console error 0 件。

**新規 tool**: `frontend/scripts/snap-error-capture.mjs` — playwright で console error / pageerror capture (visual harness exception 4 条件遵守)。

### v107 hotfix 教訓 (最重要)
- **早 return より前に全 hooks を置く** (Rules of Hooks)

- **headerCollapsed state は store keep + component 側で false 固定**: localStorage persist の migration risk 回避 + button 削除を両立する妥協 pattern
