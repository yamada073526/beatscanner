### 0-B. §0-B Phase G Phase 2 (Phase 1 keep verdict 後、 60-90 分)

Phase 1 で sub-component を**包む**だけだったが、 Phase 2 では sub-component の**内部 Card frameless 化**に進む。 二重 wrap (外: unified section / 内: Card border) が冗長に見える issue を解消。

実装方針 (v99 末で設計 draft 完了):
1. **Card primitive 拡張** [frontend/src/features/judgment/primitives/Card.jsx](frontend/src/features/judgment/primitives/Card.jsx):
   ```jsx
   export default function Card({ as: Tag = 'section', className = '', frameless = false, ...rest }) {
     const cls = frameless ? `ds-card-frameless ${className}` : `bs-panel ds-card ${className}`;
     return <Tag className={cls} {...rest}>{children}</Tag>;
   }
   ```
2. **新 CSS class** `.ds-card-frameless`: `background: transparent; border: none; box-shadow: none; padding: 0;`
3. **sub-component で frameless prop pass-through**:
   - Hero.jsx: Card 子要素として `frameless` 受け取り、 Card に pass
   - KpiStrip.jsx: 外側 .bs-panel 相当を `is-frameless` 条件付き class で切替
   - TriageBanner: `.triage-banner` 直下の bg/border を `:where(.is-frameless)` で transparent
   - FiveConditionsCard: 内部 Card に `frameless={true}` pass
   - SummaryBrief: `.summary-brief-section` の bg/border 同上
4. **UnifiedJudgmentSection 内で frameless mode 適用** (v2 mode):
   ```jsx
   <Hero ... frameless={true} />
   <KpiStrip stats={kpis} frameless={true} />
   <TriageBanner ... frameless={true} />
   <FiveConditionsCard ... frameless={true} />
   ```
5. **sub-component 間 divider 強調**: `.judgment-chapter-body > * + *` で 1px gold-fade hairline、 「章内の節」 感を出す

注意点:
- Card primitive は他 (ProfileCard / GuidanceCard 等) でも使われるが、 default false で完全に backward compat
- frameless mode は v2 内のみ → revert path 安全
- `.ds-card-frameless` は新規追加で、 既存 `bs-panel ds-card` rules と独立 → 発光バグ regression risk 低い
- 内部 padding は frameless 化で 0 になるので、 UnifiedJudgmentSection 内 `gap: var(--space-5, 20px)` が breathing room を担保

## 2. 触ると危険な箇所 (CLAUDE.md + v98 + v99 教訓)

- **FMP API path Legacy 認定**: `/api/v3/*` `/api/v4/*` は 2025-08-31 以降 403 → `/stable/*` 移行
- **FMP `/stable/stock-peers` の response 形式変更**: 旧 `{symbol, peersList:[]}` → 新 `[{symbol, name, ...}]`
- **FMP 部分確定 data**: 最新四半期で 1 segment のみ確定 (NVDA Q1) → 「2+ segments 揃った Q」 選択ロジック必須
- **vision-eval frame 8 統一** (G-4 で対策済、 ticker 切替で frame 数変動 → motion 軸基準 ばらつき防止)

### v99 新規教訓
- **demo IP rate limit**: `/api/profile-summary/<T>` が 3 req/IP/day 超で `{"detail": "..."}` を返す → regression と誤認しがち。 smoke test で 0 segment が出たら必ず `keys` で確認
