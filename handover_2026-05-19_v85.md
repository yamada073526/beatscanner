## 3. v85 で確立した永続教訓 (CLAUDE.md 昇格候補)

### 教訓 1: PGE 3 体ループの Generator early stop 3 件発生

Sprint 4 (SectionDivider) / Sprint 5 (condition pulse) / Polish Sprint 1 (multi-review + Item 3+6) で Generator が早期停止し parent 補完が必要だった。

**原因推定**: Generator session length 限界 + task 複雑度 (3 件以上同 sprint)。

**対策**:
- 1 sprint = 2 件以下に分割推奨
- Generator 起動時に「全 step を 1 session で完了」 を明示 (v85 中盤以降は明示で改善)
- 続行 Generator (`SendMessage` to agentId) は schema 未提供のため、 早期停止時は新 Generator + 明示「未完作業を完了」 prompt が現実的

### 教訓 2: Premium / Pro plan 判定の enum 漏れ pattern

`isPro={plan === 'pro'}` のように **明示的 enum 比較 で `premium` 漏れ** は 1 行 fix で済むが silent fail で長期残置 risk。

**横展開チェック**: `grep -rn "plan === 'pro'" frontend/src` で全箇所確認、 Workspace.jsx 以外にも残っていないか次セッション最初に確認推奨。

### 教訓 3: silent fail 廃止の watchlist 横展開

v84「集約 banner は silent fail 禁止」 を addToWatchlist にも適用。 今後の全 user 操作系で「click → 何も起きない」 を撤去する design principle として横展開。

`feedback_data_completeness_guard.md` の「3 段階分岐」 が「全 user 操作 silent fail 禁止」 へ拡張される pattern。

### 教訓 4: worktree merge 後の docs/references/ 反映漏れに注意

SPEC_pane3-header-polish Sprint 3 で `design_system.md` / `elevation_scale.md` に追加した token が worktree でなく main の修正だったため、 commit 7ea869c に含まれず別 commit (fab8497) で反映。

**対策**: worktree で SPEC 内 docs/references/ 修正する場合は worktree でも main でも整合確認。
