### 1.3 R5 sticky バーの試行錯誤履歴（重要・再発防止用）

| Try | 設計 | 失敗原因 |
|---|---|---|
| v1 | `blur(12px)` + 75%透過 | 透過感が弱い |
| v2 | `blur(40px) saturate(180%)` 強化 | 改善も border 強すぎ |
| v3 | カード型（左右 margin、border-radius 16px） | 左右エッジに blur 切断発生 |
| v4 | フル幅 + mask-image 上下フェード | 左右切断は解消も「箱」感 |
| v5 | `mask-image` で下端のみフェード | 同上 |
| v6 | フェード div 別 (absolute, top:100%) + linear-gradient | `transparent` キーワードで黒側補間 → 水平ライン visible |
| v7 | `rgba(...,0)` で同色α=0、+多層グラデ | まだ若干違和感、padding が「箱」化 |
| v8 ✅ | **solid bg + `backdrop-filter: saturate(180%) blur(20px)` + 1px border-bottom** | **Apple/Linear 方式で安定。フェード/mask は使わない** |

→ **教訓**: blur のフェード境界を CSS で消そうとしてはいけない。Apple は **1px border で意図的に区切る** 設計。


### 2.4 Supabase 設定（必須）

## 3. 重要な設計判断（守るべきルール）

### 3.1 デプロイ運用

- **`railway up` 以外のデプロイ手段は使わない**（preview server は不可）
- ローカルビルドでの確認は OK：`cd frontend && npm run build`
- デプロイ後の検証は本番バンドルを curl で取得 → grep で文字列確認
- 反映確認は CSS or JS のハッシュ変更で判定（Monitor ツール推奨）

### 3.2 sticky 検索バー（再変更注意）

- `backdrop-filter` の境界切断問題で 8 回試行錯誤した経緯あり
- **現在の安定版は Apple 方式**: `rgba(var(--page-bg-rgb), 0.72)` + `saturate(180%) blur(20px)` + ライト時は `border-bottom: none`、ダーク時は `1px solid rgba(255,255,255,0.08)`
- 構造: フル幅 (`marginLeft/Right: calc(-50vw + 50%)` + `width: 100vw`) + `padding: '12px 20px'`
- **触らない方が無難**

### 3.3 ヘッダー grid 構造

- 2 カラム (auto 1fr) on mobile / 3 カラム (1fr auto 1fr) on md+
- 中央タブは `hidden md:flex`
- ハンバーガーは `flexShrink: 0` で右端固定

### 3.4 「じっちゃま」表記

- UI には出さない（クラス課題提出物のため個人名を回避）
- 内部資料（コード/comment/CLAUDE.md/docs/）は残置 OK

### 3.5 タブ key の混在

- 内部 key: `'home' / 'judgment' / 'report' / 'チャート'`（最後だけ日本語）
- 文字化け回避のため変えない

### 3.6 `useScrollBlur` フック

- 一時 sticky bar の動的 blur に使用していたが廃止
- ファイルは残置（未使用）。次セッションで削除しても OK

## 8. AI 運用ルール（v23 から継承 + 補足）

1. ユーザーから具体的な仕様 → 該当箇所を編集 → `railway up` でデプロイ → 本番バンドル grep で検証
2. **`railway up` 以外のデプロイ手段は使わない**（preview server / npm run dev は不使用）
3. 編集前に `grep -n` で対象行を特定。広範な編集は `replace_all=true` を活用
4. デプロイ後は新バンドルを curl 取得 → 文字列 grep で反映確認
5. CSS の minify で変数名/小数表記が変わるため、grep は柔軟に
6. ローカル `npm run build` で構文エラーは事前に弾く
