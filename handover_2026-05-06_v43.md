## 4. 本セッションの主要実装詳細

### 4-1. 翻訳 SSE 化（コミット `7a3f170`）

**Backend**: `/api/translate/stream` 新設（既存 `/api/translate` は温存）
- キャッシュヒット分は最初に即時 emit（TTFT ~0.1s）
- 未キャッシュ分は Claude Haiku の `stream_complete` を行単位パース → 番号行確定ごとに emit
- 取得漏れは原文 fallback で必ず emit（フロント欠損防止）

**Frontend**:
- `api.js`: `translateTextsStream(texts, onItem, signal)` 追加
- `hooks/useTranslation.js`: 「全件揃ってから setTranslated」を撤廃 → 初期 merged 配列 (cached + 原文 fallback) を即時 set → SSE で 1 件届くごとに `setTranslated([...merged])` 逐次差し替え

体感: 3-4s 待機 → 0.1-0.5s で先頭タイトル切替

## 5. 🔴 重要知見 — 永続化必須

### 5-1. Supabase migration の手動実行は SQL 2 ファイル順次（v42 §5-2 適用）
- Supabase SQL Editor で作成したテーブルは GRANT が別途必要
- `docs/migrations/2026-05-06_holdings_x2.sql` → `docs/migrations/2026-05-06_holdings_x2_grants.sql` の順で実行
- `grant select, insert, update, delete on public.holdings to authenticated;` を忘れない

### 5-2. FMP 無料プランの daily limit と /api/quotes フォールバック
- FMP 無料: 250 req/day、`/quote/{symbols}` bulk は paid only の場合あり
- `/api/quotes` は **yfinance fallback で堅牢化済**（market-indices と同パターン）
- 502 を返さず空配列を返す設計 → フロント Watchlist は「…」バッジで loading 表現
- リリース前 FMP 有料契約 ($14/月) で primary が FMP に戻り、コード変更ゼロで自動高速化

### 5-3. modal の `.modal-panel` クラス必須（HoldingModal バグから学習）
- 旧 HoldingModal が `.modal` という未定義クラスを使い背景貫通バグ発生
- 既存 `.modal-panel` (background + border + shadow + animation) を使うべき
- title は `<h3>` ではなく `<h2>`（`.modal-header h2` セレクタとの整合）
- 案 D で HoldingModal は完全削除済だが、将来モーダル新設時の落とし穴として記録

### 5-4. PnL バッジが見えない問題の診断手順
- chip rendering: `holdings[t]` AND `prices[t]` が両方 truthy で初めて badge レンダリング
- どちらか欠落 → silent fail
- `holdings.shares` / `holding[t]` が undefined のまま → useHoldings の Supabase fetch が走っていない（RLS / GRANT を疑う）
- prices が undefined → /api/quotes が 502 (FMP 障害) 返している → yfinance fallback の確認
- デバッグには「holding あり + price なし」時に「…」灰バッジを出す設計が有効

### 5-5. 4 体エージェント並列レビューのコスト感
- 1 体あたり ~30-60 秒、4 体並列で 1-2 分で多視点意見が揃う
- プロンプト内に「設計思想 5 原則」+「現状 UI 構造 (wireframe)」+「ユーザー指摘 (要望)」+「観点 5 つ」+「出力フォーマット」を必ず含める
- Web 検索が拒否された場合は「業界知識ベース」と明示してもらう（金融 / UI/UX レビューはこれで十分機能した）

### 5-6. UI 階層問題の解決パターン（universe → subset, parent → child）
本セッションで 2 度遭遇:
1. ウォッチリストフィルタ（モード × タグ並列が親子に見える） → カード見出しに universe 件数昇格 + Notion 暗黙化
2. HoldingSection のタブ + 入力（並列に見える） → カード化 + tabpanel surface + slideDown animation

**共通解**: **「surface 階層を 1 段下げる」** がモダン UI の標準（Linear / Notion / shadcn-ui / Apple HIG）。インデントは モバイルで破綻しがちなので避ける。
