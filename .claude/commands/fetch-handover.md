# /fetch-handover — 最新引き継ぎ書の取得・要約

新セッション開始時の context warm-up 用。最新の `handover_YYYY-MM-DD_v*.md` を 1 つだけ読み、要点を圧縮して返す。

## 実行手順

### 1. 最新ファイルを特定
```bash
ls -1 handover_*.md 2>/dev/null | sort -V | tail -1
```
これで「最も新しいバージョン番号」のファイルを得る。

### 2. ファイル全体を Read で読む
Read tool で取得 (引き継ぎ書は通常 200-400 行)。

### 3. 以下の項目を抽出して 30 行以内のサマリーで返す

```
📋 引き継ぎ書 v?? サマリー (作成日 YYYY-MM-DD)

🎯 完了済みの主要トピック (3-5 行):
  - ...

🔴 最優先 — 次セッション着手必須 (引き継ぎ書 §2 から):
  - ...

🟡 重要知見 (永続化要、引き継ぎ書 §5 から、特にセッション固有のもの):
  - ...

📊 残バックログ (未完 Phase / 工数):
  - X-2-5-? : 内容 (工数)
  - ...

⚠️ 触ると危険な箇所のリマインダ (CLAUDE.md からも引用):
  - sticky 検索バー
  - VITE_ARG/ENV 同期
  - ...
```

### 4. 旧バージョン (v??-1 以前) は読まない

CLAUDE.md ルール:「賞味期限は 1〜2 セッション。古くなったら削除して構わない」
古い handover を読むのは context window の浪費。最新版のみで十分な情報が得られる設計。

## 注意

- 引き継ぎ書本体を Read で開いて読むこと (grep でつまみ食いしない)
- 全文要約せず、**「次セッションが知っていれば困らない」** 情報に絞る
- 該当ファイルがなければ「handover ファイルが見つかりません」と返す
