## 🟡 重要知見(永続化候補)

- **monorepo の本番検証は CI に委ねる前提**: 本環境は egress policy で `beatscanner-production.up.railway.app` が 403(retry 禁止)。**snap/curl/dogfood は本番 access のある環境か post-deploy CI でしか走らない**。代替として **exported 純関数(itemPasses/buildActiveGrades/countPreset)を esbuild バンドル → Node で offline unit test** が count==list の ground-truth 検証に有効(本セッションで確立)。
- **監査の訂正**: vol(volume_surge_pct)/inst(inst_holders_qoq_pct)は「効かない crow」ではなく **default OFF の動作するトグル**(非 core grade は off 初期・クリックで override 追加され適用)。真の問題は §0 の hidden core grade(crow 非表示のまま暗黙 AND)だった。
- **§0 の根**: `applyStrategyImpl` が `setPreset('standard')` するため `buildActiveGrades` が全 preset で PRESET_CORE_KEYS を適用 → new_high_break 等で eps/cagr/roe が hero=0 を招いていた。preset 別 grades 宣言で解消。
