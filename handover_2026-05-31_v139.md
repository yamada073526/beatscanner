## 🛠 本セッションの教訓 (運用 SSOT、 次セッションで踏まないこと)

1. **ProTeaser は共有 component で独立 chunk (`ProTeaser-*.js`)**。 LP の変更検証は「grid 系=LandingPage chunk /
   ProTeaser 系=ProTeaser chunk」 と分けて grep。 LP chunk だけ見て「変更が乗ってない」 と誤判定しがち。
2. **LandingPage には grid div が 6 個** (Today notable / Earnings soon / Recent / sample 等 + FeaturesSection 1080 +
   PricingSection)。 maxWidth/minmax を編集する時は **child 内容で対象 grid を特定** (FeaturesSection は
   `alignItems:'stretch'` + FeatureCard、 PricingSection は「Free プラン」 コメント)。 一括 grep 置換は誤爆する。
3. **Edit の old_string は実ファイルを Read してから**。 記憶や前回出力 (garble する場合あり) でなく現物 copy。
   本セッションで「String not found」 失敗が多発、 失敗 Edit を成功と誤認しないこと。
4. **Bash tool に `output` パラメータを付けない** (`command` のみ)。 誤パラメータで parallel batch が全 cancel される。
