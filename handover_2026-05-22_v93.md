### Phase 2.8 で着地した 5 件 (root cause 確定済)

#### #1 アナリスト視点 左右クリッピング fix + #5-a 会社概要クリッピング 3 回目 fix (1 つの root cause で 2 件解消)

- **root cause 確定**: `AccordionSection.module.css` の `.panel { overflow: hidden }` rule が framer-motion m.div の state-aware `style={{ overflow: isOpen && !isAnimating ? 'visible' : 'hidden' }}` を CSS specificity で上書きしていた
- Phase 2.6 で root `.section` の overflow:visible 化 → 子 `.panel` は hidden のまま放置で効果なし
- Phase 2.7 で m.div に inline style 追加 → CSS specificity で負けて効果なし
- Phase 2.8 で `.panel` CSS rule の overflow:hidden を削除 → JS state-aware が有効化、 #1 と #5-a が同時解消
- **メタ教訓**: 「複数 dogfood feedback が同じ chain bug」 の pattern 認識 ([[feedback-clipping-root-cause-chain]] に起票済)

#### #2 GuidanceCard 発光本格強化 (user dogfood 4 回目「まだ弱い」 解消)

- inset shadow opacity 20% → 30%
- outer `0 2px 8px 0 accent 12%` shadow 追加で EarningsHistoryChart cluster と同等 elevation
- `[data-testid="guidance-card-wrapper"].tier-m-glow` に border tint 追加 (light: 25%, dark: 32%)
- token 経由のみ (`color-mix(in srgb, var(--color-accent) N%, ...)`、 hex 直書き禁止)
- contain: paint 禁止維持

#### #3 5 section halo accordion 連動 (IO trigger 単独だと collapsed で不発)

- **横展開 SSOT**: 将来 accordion / collapse / tab で section 隠す UI に halo 追加する pattern ([[feedback-halo-sweep-accordion]] に起票済)
