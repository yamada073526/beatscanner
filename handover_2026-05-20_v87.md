## 1. PGE 3 体ループ自律運用の実証 (v87 で確立)

### user 10h 留守中の PDCA loop 動作

v86 で SSOT 化した PGE 3 体ループ (planner → generator → evaluator) を **user gate 1 + gate 2 省略** で稼働:
- gate 1 (Planner 後の SPEC 承認) 省略 → BS_AUTONOMY_MODE=1 で Generator 自動起動
- gate 2 (Generator 後の commit/deploy 承認) 省略 → main consolidate + commit まで自律
- L4 主観 gate (multi-review) は **Phase 完了時に 1 回集約** (sprint 毎は不要)

**ペース実績**: Phase 1 (8-12 人日見積) を **約 1.5h** で完走 (3 sprint × 平均 25-30 min + 3 体合議 + hotfix)。

### v86 落とし穴 4 件への対応

| 落とし穴 | v87 での回避策 | 結果 |
|---|---|---|
| worktree 非累積 | 各 sprint 完了時に worktree → main merge を Bash で実行 | sprint 2 が sprint 1 merge 済 main から worktree 派生されたことを確認 |
| selector hallucination | 新規 testid (`condition-sparkline-N` / `cfps-eps-delta-Q{N}` / `diagram-section-{type}`) を source + dist 両方で grep verify | 全 3 sprint で dist match 確認、 hallucination 0 件 |
| ESM top-level return | snap script 配置なし (vision-eval 不使用) | 該当なし、 risk 0 |
| infinite animation finish() | 全 Recharts Line / Bar / ReferenceLine に isAnimationActive={false} を明示 | 3 sprint 全 PASS |

### Generator subagent の制約発見

Generator subagent は **design-system-check までは PASS report する**が、 self-eval JSON 出力 + Evaluator 自動引き渡しを毎回完遂しない (Sprint 1 / Sprint 3 で発生)。 main 側で手動補完が必要:
- build verify (`npm run build`) 手動実行
- testid grep verify 手動実行
- NaN safety (`MNaN` 0 件) 手動 grep
- Evaluator subagent を手動起動 (`Agent(subagent_type='evaluator', ...)`)

これは Generator subagent definition の改善余地 (.claude/agents/generator.md の Step 5-6 を強化推奨)。
