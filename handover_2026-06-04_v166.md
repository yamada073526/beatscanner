## 🔴 今晩のオートパイロット 最優先タスク — ASO 等「年次3期未満」銘柄の分析失敗 (user dogfood 2026-06-04)

**症状**: スクリーナーから ASO 選択 → 「判定待ち / 取得待ち」 + ファンダ5条件が「分析中…」 のまま固まる、 図解 (図) も生成不可。

**真因 (本セッションで切り分け済)**:
- `curl /api/analyze/ASO` → **HTTP 422 `{"detail":"Need at least 3 annual periods of data, got 1"}`**。
- ASO (Academy Sports) は 2020 年 IPO = 本来 ~5 期分あるはず → **backend の年次データ取得が過少 (got 1)**。 FMP fetch の endpoint/plan 制約 or 期間カウントロジックの bug が疑い。
- 加えて **frontend が 422 を graceful 処理せず無限「分析中…」** = Trust Cliff (壊れて見える)。
- 図解 (`/api/visualize/{ticker}`) も同じ分析データ依存で失敗していると推測 (要確認)。

**autopilot がやること (原因究明 + 復旧)**:
1. backend: なぜ ASO の annual periods が 1 しか取れないか。 `/api/analyze/{ticker}` の年次データ取得経路 (FMP income statement 等) を追い、 ASO で実際に何期返るか・3期未満の判定箇所を特定。 他にも 3期未満で同様に落ちる銘柄がどれだけあるか調査 (近年 IPO 銘柄)。
2. backend fix: データが本当に取れるなら取得修正。 取れない (genuine に新規上場) なら **422 でなく「データ不足で一部分析不可」 を構造化 response で返す** (sources schema / partial 方針、 [[feedback_data_completeness_guard]])。
3. frontend fix: 422/データ不足を graceful に — 無限「分析中…」 をやめ「年次データ不足で5条件は評価できません」 等の明示表示 (skeleton 固着回避)。 図解も同様に「データ不足」 表示。
4. hallucination-guard / content-audit skill 経由で検証、 §38/景表法 safe な文言に。
5. 検証: ASO + 他の近年 IPO 銘柄 数件で再現/解消を curl + (可能なら) snap で確認。

⚠️ **触る前**: `hallucination-guard` skill (visualizer/aggregator 触るなら 4層防御)、 LLM 数値計算 BAN。 aggregator は数値物理層。
