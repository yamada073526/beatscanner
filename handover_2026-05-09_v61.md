## 4. 🟡 v61 で確立した重要知見

### 4-1. Design System 5 基準 (Aman/Ritz-Carlton 級の操作的定義)
1. Hover Ring 強度: arrival 0.36 / hover 0.60 (light) / arrival 0.50-0.62 / hover 0.75-0.80 (dark)
2. Stagger 順次遅延: 40ms × n、8 件で総 320ms 以内
3. Font Weight Contrast: Stat fw700 : Label fw500 (1.4×)
4. Line-height 比率: Stat ≤1.1 : Label ≥1.3 (1.18× 以上)
5. Focus 区別: Hover=cyan / Focus-visible=gold + 2px white outer

### 4-2. v2 ticker sync の挙動
- `currentTicker` prop (App.jsx の `ticker` state) を JudgmentTab に渡す
- `useEffect([currentTicker])` で `selectTicker(currentTicker)` を発火
- 意図的に `selectedTicker` を deps に入れない (手動切替後に巻き戻り防止)

### 4-3. PremiumLock の API
```jsx
<PremiumLock feature="..." plan={plan} label="..." bullets={[...]} onUpgrade={fn}>
  <Card>...</Card>
</PremiumLock>
```
- `canUse(feature, plan)` で透過判定
- `onUpgrade` 未指定なら static lock (BC)、指定で CTA ボタン

### 4-4. Hook 1 (pre_edit_design_tokens.sh) の挙動
- 対象: `frontend/src/**/*.{css,jsx,tsx,js,ts}` の Edit/Write
- `new_string` にあって `old_string` にない raw hex (`#xxxxxx`) を block
- whitelist: `docs/references/elevation_scale.md` の `ALLOWED-HEX:` 行
- 3 文字 hex (`#fff`) と docs 系は対象外

### 4-5. 削除しない / 触らない箇所 (新規)
- **`?j1=1` の v1 fallback path**: dogfood QA まで保持. v1 component 群 (ResultBadge / ConditionCards / GuidanceCard 直 render etc) は v1 のために残す
- **`features/judgment/state/useJudgmentResult.js`** の signature: App.jsx の他箇所が依存. setter 削除禁止
- **`prefetchedRef` を hook より前に置く理由**: hook が prefetch コールバックを必要とするため. 移動禁止
