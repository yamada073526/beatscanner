## 2. 🔴 重要知見 — 永続化必須 (memory にも記録済)

### 2-1. React useEffect の dep 暴発パターン (今セッション最大の知見)

**症状**: NotificationSettingsModal で focus trap が機能しない / ESC で閉じない。

**原因**: 親 (App.jsx) が `onClose={() => setShowNotifModal(false)}` を**インライン生成**。
毎レンダで新参照になるため、子の `useEffect(..., [isOpen, onClose])` が**親再レンダ毎に再評価**:
1. cleanup 実行 → `previouslyFocusedRef.current.focus()` で開く前の要素 (ハンバーガー) へフォーカス強制復元
2. effect 再実行 → 新しい `previouslyFocused` を捕捉 → setTimeout(0) で modal 内へフォーカス

App は価格ポーリング (15s) や watchlist sync で頻繁に再レンダするため、この cleanup→re-run が**秒単位で繰り返される** → focus が定着できない。同様に keydown listener も剥がれるタイミングで ESC が取りこぼし。

**修正パターン (確立)**:
1. **2 つの effect に分離**:
   - 焦点管理 effect: `[isOpen]` のみ依存 → 真の open/close 遷移時だけ走る
   - キーハンドラ effect: `[isOpen]` のみ依存 (onClose は除外)
2. **`onClose` は ref 経由参照**:
   ```js
   const onCloseRef = useRef(onClose);
   useEffect(() => { onCloseRef.current = onClose; }, [onClose]);
   // handler 内では onCloseRef.current?.()
   ```
3. **previouslyFocused はクロージャ変数で保持** (ref でなく)
   - effect スコープ内で const として → effect 寿命と一致

**今後のモーダル実装はこのパターンを必ず踏襲すること** (TagManagerModal も同様の構造で潜在バグあり、要監視)。

### 2-2. 経済カレンダー静的フォールバックの曜日範囲ガード

`_generate_static_economic_events` (backend/app/main.py) の月次指標は曜日 × 日数範囲で複数日マッチしうる:
- CPI: `weekday in (1, 2) and 9 <= day <= 14` → 火・水両方マッチ
- Core PCE: `24-31` 範囲は金曜が 2 個入りうる
- ISM Services: `3-5` 範囲は最大 3 日マッチ

**修正パターン**: `_emitted: set[(year, month, event_key)]` + `_emit_once()` ヘルパで月 1 回保証。週次イベント (Initial Jobless Claims) はガードしない。

**TZ 問題**: `_iso_at()` が naive datetime を返していたため、フロント `new Date()` がローカル TZ として解釈 → US 8:30 ET (JST 22:30) が JST 13:30 と表示。`isoformat() + "Z"` で UTC 明示し解消。

**FMP 有料化で解決**: `if not raw: raw = _generate_static_...` のフォールバック構造のため、有料化後は本関数が走らなくなる。今回の修正は移行までの過渡的安全網。

### 2-3. 色トークン化のパターン (Web 設計レビュー #7)

**Before**:
```css
.foo-gain { color: #16a34a; }
.foo-loss { color: #dc2626; }
[data-theme="dark"] .foo-gain { color: #34ef81; }
[data-theme="dark"] .foo-loss { color: #f87171; }
```

**After**:
```css
:root {
  --color-gain: #16a34a;
  --color-loss: #dc2626;
}
[data-theme="dark"] {
  --color-gain: #34ef81;
  --color-loss: #f87171;
}
.foo-gain { color: var(--color-gain); }
.foo-loss { color: var(--color-loss); }
```

→ dark theme override が不要に、CSS が 0.6KB 削減 (86.78KB → 86.19KB)。新コンポーネントは自動でテーマ追従。

**chart 系 (lightweight-charts)** は hex 文字列を要求するため CSS 変数を直接使えない → `CHART_PALETTE` 定数を JS にも保持して同期。

### 2-4. amber 色は「警告・緊急」専用 (CLAUDE.md 永続ルール再確認)

経済カレンダー Beat/Miss 矢印が amber を使っていたが、HIGH impact pill / TODAY バッジ / 最注目バッジと混同される。さらに「CPI 上振れ = インフレ加速 = 株価ネガ」のように経済指標は単純に上=良くない → **価値中立な slate** に変更。コンポーネント既存コメントの設計意図 (「中立的なシアン/グレー」) と一致させた。

### 2-5. accessibility パターン: aria-invalid + role="alert"

**フォームバリデーション**:
1. `errorField: 'shares' | 'price' | null` で「どの input が invalid か」を記録
2. 該当 input に `aria-invalid="true"` + `aria-describedby={errorId}`
3. エラー <p> に `role="alert"` + 一意 `id={errorId}`
4. CSS で `[aria-invalid="true"]` に赤ボーダー + 微妙な背景 + focus 時 ring
5. 入力変更で `aria-invalid` を即時クリア (二重通知回避)

**saving 中ボタン**:
- `aria-busy={saving}`
- 12px インライン SVG + `@keyframes spinner-rotate` 0.8s linear infinite
- 「保存中...」「削除中...」テキストと併用

- **投資業界の色ルール厳守**: 緑=上昇 / 赤=下落 / amber=警告のみ / シアンはブランド色 (上昇でない)
- **`railway up` は worktree から実行不可** — main repo から `git merge --ff-only` 後にデプロイ (v44 で判明、memory 永続化済)
- **モーダル useEffect dep に inline onClose を入れない** — v45 で判明 (本書 §2-1)、今後のモーダル実装は ref + 2 effect 分離パターン必須
