# UXレビュー beatscanner v19

レビュー日時: 2026-04-29  
対象バージョン: main ブランチ最新（図解生成インライン表示 実装後）  
参照画像: PC ダークモード（MSFT）+ スマホ ダークモード（AA）×2

---

## 🔴 重大（すぐ直すべき）

### 1. 成長ストーリーSVGがスマホで読めない
- **問題**: `viewBox="0 0 800 ..."` + `width:100%;height:auto` の組み合わせで、390px幅スマホでは0.49倍スケールダウン。11pxのSVGテキストが**実効5.4px**になる。グラフの数値・期間ラベルが実質判読不能。
- **理由**: SVGの `viewBox` はユーザー座標系を固定するため、レスポンシブスケールに従いすべてのフォントが比例縮小される。
- **改善案**: `buildGrowthSVG` のコンテナに `minWidth: '600px'` を設定し、`overflowX: 'auto'` でスクロール対応にする。或いは viewBox を `0 0 400 ...` に変更し 2×2 → 1列レイアウトへ切り替える。

### 2. タブバーのタップターゲットが 44px 未満
- **問題**: `py-2`（上下 8px padding）+ `text-sm`（14px）= 実効約 **30px** 高さ。Apple HIG・Material Design ともに最小 44px を推奨しているが下回っている。
- **理由**: `App.jsx:548-595` のタブボタンが Tailwind の `py-2` のみで高さを制御しており、スマホタップ時に誤タップや連打が発生しやすい。
- **改善案**: `py-2` → `py-3`（12px padding → 実効38px）以上に変更、または `min-h-[44px]` を追加する。

### 3. タブバーに CSS `hover:` が残存（プロジェクトルール違反）
- **問題**: `App.jsx:554,564,577,588` に `hover:text-slate-700 hover:bg-slate-200` が記述されている。スマホのタッチ操作で hover 状態が残留し、タップ後もハイライトが消えない。
- **理由**: プロジェクトルールではホバー演出は `onMouseEnter/Leave` で実装し、Tailwind の hover クラスは使わないとされている。
- **改善案**: `hover:*` クラスをすべて削除し、アクティブ判定は `activeTab === xxx ? 'bg-white...' : ''` の条件分岐だけに絞る。

## 🟡 中程度（次のスプリントで対応）

### 4. 強み・リスク 2カラムがスマホで窮屈
- **問題**: スクリーンショット3でも確認できるが、`grid-template-columns: 1fr 1fr` の 2カラムは 390px 幅では各カラム約 185px になり、25字以内のテキストが3〜4行折り返される。
- **理由**: 日本語は1文字あたり約13〜14px（12px font）のため、185px 内に25字を表示すると折り返しが多発し縦に伸びすぎる。
- **改善案**: スマホでは `grid-template-columns: 1fr`（1列）に切り替え、強み・リスクを縦積みにする。`@media` の代わりに `window.innerWidth < 480 ? '1fr' : '1fr 1fr'` で対応。

### 5. 「生成完了」後にVizPanelが追加されてもスクロールが必要でユーザーが気づかない
- **問題**: ボタンクリック → ローディング → 「生成完了」バナー表示 → VizPanel が下に展開されるが、ユーザーはローディングバナーの位置を見ているため、VizPanel が増えたことに気づかない場合がある。
- **理由**: 生成完了時に自動スクロールやアニメーションがなく、パネルが静かに追加される。
- **改善案**: 生成完了後に `vizPanelRef.current?.scrollIntoView({ behavior: 'smooth', block: 'start' })` を呼び出してVizPanelまでスムーズスクロールする。

### 6. ビジネスモデルフローのdetailテキストがオーバーフローする可能性
- **問題**: `buildFlowSVG` でボックス幅 136px、detail テキストを1行固定で描画している。Claude が15字以上のテキストを生成した場合、ボックスをはみ出す。
- **理由**: SVGの `<text>` は自動折り返しをしないため、文字列が長ければはみ出しになる。
- **改善案**: detail テキストを `textLength` で圧縮するか、9〜10字で切り捨てて省略記号を付ける。または `foreignObject` を使って HTML テキストの折り返しを活用する。

### 7. SVG保存ボタンがスマホでは小さい・気づかれにくい
- **問題**: 右上の「SVG保存」ボタン（padding `5px 10px`）はスマホで幅約70px・高さ約28px。44px 未満かつ端に寄りすぎており、タップミスしやすい。
- **理由**: 右上配置はスマホ持ち手では届きにくい「死角」エリア（右上サム領域）に位置する。
- **改善案**: ヘッダーではなく VizPanel の**最下部**にフル幅ボタン（`width:100%`）として配置し直す。または上部ヘッダーのまま維持するなら `padding: '10px 16px'` に拡大する。

## 🟢 良い点（維持すべき）

- **ヘッドラインの存在感**: 28px 大型テキストで企業の状況が一目でわかる。「AI・クラウド駆動で全指標右肩上がり」のようなコピーはスキャニングに最適。
- **PASS/FAIL バッジの視認性**: 緑/赤の明確な色分けと `passCount/totalCount` 表示が判断をサポートしている。ダークモードでも十分なコントラスト。
- **ビジネスモデルフローの矢印UI**: ボックス→矢印→ボックスの構造が直感的。sky blue (#38BDF8) の統一カラーがアクセントとして機能している。
- **セクションの情報階層**: ヘッドライン → ビジネスモデル → 数値 → 強みリスク → 問い、という流れが「概要→詳細」の認知モデルに合っている。
- **ダークモード完全対応**: CSS変数 (`--bg-primary`, `--text-primary`) の一貫した使用と `data-theme="dark"` の適切な切り替えが機能している。
- **「投資家への問い」セクション**: 左ボーダー + テキストの構成が読みやすく、分析の結論として機能している。
