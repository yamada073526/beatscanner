## 🟡 重要知見 (永続化候補)

### useMemo + ref の永続化罠 (R7-R9 教訓)
- useMemo deps が同じなら memo が同 components を返す → table fn closure 内 ref が永続化
- 2 回目以降 render で「ref.current=true」 残ったまま固定 → first-time-only logic が壊れる
- 解: useMemo 廃止 (毎 render で再生成、 perf cost 微増だが feature 優先)

### React Fragment vs div display:contents
- ReactMarkdown の custom component で Fragment `<>...</>` で兄弟 element を返す時、
  table 等の特殊 element の child として扱われ DOM stripped 可能性
- 確実化: `<div style={{display:'contents'}}>` で flow layout 維持

### <details> marker 全 vendor 抑止
- Chrome: `::-webkit-details-marker { display: none }`
- Firefox / Safari / 標準: `::marker { content: ''; display: none }` 両方必須
- list-style: none も併用で確実消去

### regex alternation precedence + m flag
- `(A|B)` 単独でなく `(?:A|B)` で非キャプチャグループ化必須
- m flag + `$` lookahead は each line end で early match の罠
- undefined access guard も併用 (`m && typeof m[1] === 'string'`)

### FMP primary AND 化
- 旧: `not income or not cash` → 片方欠落で yfinance fallback (中国 ADR / SMCI で空に落ちる)
- 新: `not income and not cash` → FMP 完全失敗時のみ yfinance、 片方取れた場合は judge() で補完

### multi-review 3 体合議 cost-efficient pattern
- background=true で並列起動、 main session で並行作業
- sonnet モデル指定で cost 5x 削減 (Opus → Sonnet)
