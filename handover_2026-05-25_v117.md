## 🟡 重要知見 (永続化候補、 本 session 教訓)

### Docker layer cache hit 問題 (v116 R0 + R6 教訓)
- frontend/ 内 source 変更ゼロの状態で `RUN cd frontend && npm run build` layer は cache hit する
- build-articles.mjs (Supabase fetch + SSG) が再実行されず、 article publish が反映されない
- **R0 短期対策**: RUN 行内に echo を埋め込み Dockerfile hash 強制更新
- **R6 恒久対策**: `ARG BUILD_TIMESTAMP=stable` 化、 Railway Service Variables で `BUILD_TIMESTAMP=$(date +%s)` を任意 set で invalidation 可能

### React component 内 regex の 2 大落とし穴 (v116 R6b hotfix 教訓)
- `m` (multiline) flag + `$` lookahead は each line end で early match (content 1 文字で extract 終了)
- alternation `(A|B)` を 単独で書くと precedence で誤 grouping (`X(A|B)` でなく `XA|B`)
- 必ず非キャプチャグループ `(?:A|B)` で囲む、 undefined access guard も併用

### remark-gfm は必須 (v116 R4 教訓)
- react-markdown 単独では GFM table syntax (`| col |---| value |`) を parse しない
- raw `|---|` テキストが `<p>` に落ちるブランド毀損バグになる
- `remarkPlugins=[remarkGfm]` 必須

### Auto-PDCA 自動検証は selector specific が肝 (v116 R6 教訓)
- `--selector "main.article-page"` は viewport 上部のみ screenshot、 scroll 範囲外は映らない
- 確認したい element の `[data-testid='...']` を直接 selector に指定すると Playwright が auto-scroll
- 「fail」 verdict も viewport 制約か実装漏れか判別必須

### company name → ticker mapping は longest-first (v116 R6 教訓)
- 「Berkshire Hathaway」 を「Berkshire」 より先に match する必要
- `Object.keys(map).sort((a, b) => b.length - a.length)` でソート後 iterate

### multi-review 3 体合議 vs 6 体合議 (v116 で 5 セット使用)
- 純 UI 改善 / readability 改善 = 3 体 (ui-designer + frontend-architect + qa-dogfooder) で十分
- 合議所要時間 ~2 min、 cost 効率 6 体の 50%
- LP 訴求 / Hallucination Guard / 新 backend endpoint は 6 体推奨 (handover v82 SSOT)

## 触ると危険な箇所 (v117 追加)

### v116 session 教訓 (新規)
- **`.article-implications` 内 extractImplications()** の regex は m flag 厳禁 + alternation は `(?:)` 必須 (R6b hotfix 教訓)
- **Dockerfile `RUN cd frontend && npm run build`** layer に依存する SSG output は Docker cache hit で失われやすい、 cache-bust ARG (BUILD_TIMESTAMP) で制御
- **ArticleHero.jsx `verdict` prop** = badge tone (緑 / 赤 / amber) の数値 anchor、 削除すると WATCH badge が消える
- **build-articles.mjs `COMPANY_TICKER_MAP`** = 副次 ticker link の SSOT、 削除すると Berkshire / Visa 等の cross-link 全消失
- **`extractTLDR / extractImplications`** = body_md pre-process layer、 これを bypass すると含意 2 列 callout が plain markdown text で render される
- **remark-gfm import** = ArticleBody.jsx の `remarkPlugins=[remarkGfm]`、 削除すると Markdown table が `|---|` raw text で表示 (R4 重大 regression)
- **ArticleErrorBoundary** = ArticlePage を wrap、 削除すると ArticleBody runtime error で全 page 白画面 (v75 chart 白画面と同種)
- **`#article-implications__panel--bull/bear/action`** = 緑 / 赤 / gold の token mapping、 設計判断時に色を変えない (投資業界色ルール厳守)
- **既存 GOOGL article body_md / subtitle / title** = R1-R6 backfill 累積、 Supabase 直編集する際は format (TL;DR + Before/After + ### 強気 etc) を維持
