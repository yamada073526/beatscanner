**変更前**: `isProUser = isPro()` (FMP API キー有無で判定する旧ロジックが混在)

## 6. Checkout 完了後ポーリング（Webhook タイムラグ吸収）

**背景**: Stripe Checkout 完了 → `/?checkout=success` リダイレクト後、Webhook が Supabase を更新するまで数秒のタイムラグがある。この間 `isSubscribed=false` のまま Pro 機能が見えず、ユーザーが混乱する。

**実装**（`App.jsx` + `useSubscription.js`）:

```js
// ?checkout=success 検知時
// 1. URL から ?checkout=success を除去 (replaceState)
// 2. 「🔄 決済を確認中...」トースト表示
// 3. 2秒間隔・最大15回(30秒)で refetchSub() をポーリング
// 4. isSubscribed=true になったら「🎉 Pro 会員になりました！」トースト
// 5. 30秒超過で「⚠️ ページを再読み込みしてください」トースト
```

`useSubscription.js` に `refetch()` 関数を追加してポーリングから呼び出せるようにした。

## 7. Webhook 修正（3件）

### 7.1 Stripe SDK v10 対応（`8552c02`）
**問題**: SDK v10 で `event["data"]["object"]` がエラーになる場合がある。

**修正**: 属性アクセス優先に変更
```python
sub = getattr(getattr(event, "data", None), "object", None) or event["data"]["object"]
```

**その他**:
- `sub.items.data` を try/except で囲み、失敗時は `_obj_get` にフォールバック
- Webhook ハンドラ全体を `try/except` で包み、例外時も **常に 200 を返す**（500 での無限リトライ防止）
- 各ステップに `print` ログ追加

### 7.2 metadata の `_obj_get` 対応（`928a8b0`）
**問題**: Stripe SDK v10 で `session.metadata` が `StripeObject` 化 → `.get()` で AttributeError。

**修正**:
```python
def _obj_get(obj, key, default=None):
    # StripeObject（属性アクセス）と dict（キーアクセス）の両方に対応
    try:
        return getattr(obj, key, None) or (obj.get(key) if hasattr(obj, 'get') else default)
    except Exception:
        return default
```

### 7.3 `current_period_end` の item レベル fallback（`d2ebe58`）
**問題**: Stripe API バージョン 2025-03-31 以降、`current_period_end` がサブスクリプションレベルから **アイテムレベル** に移動。

**修正**:
```python
cpe = _obj_get(sub, "current_period_end")
if cpe is None and first_item is not None:
    cpe = _obj_get(first_item, "current_period_end")
```

## 14. 指標バー拡充 + 時事ニュース セクション 設計案（v41 候補）

> 本セッションで 3 つの専門家サブエージェント（金融アナリスト / Webアプリ設計 / UI/UX）にレビュー依頼し、以下の設計方針を確定。実装着手前にユーザー最終承認を得ること。

### 14-1. 指標バーに追加すべき指標（金融アナリスト視点）

**最低限追加 5 件（Tier 1）**:
| ティッカー | カテゴリ | 役割 | データ |
|---|---|---|---|
| `DX-Y.NYB` (DXY) | 為替 | ドル全体強弱（USD/JPY より上位概念） | yfinance |
| US2Y / イールドスプレッド (US10Y - US2Y) | 金利 | 逆イールド = リセッション先行指標 | FRED API 推奨 |
| HYG | クレジット | ハイイールド債、リスクオフ初動（VIX より先行） | FMP / yfinance |
| TLT | 債券 | 長期米国債、グロース株との相関逆転シグナル | FMP / yfinance |
| `CL=F` (WTI) | コモディティ | インフレ・地政学・エネ直撃、中東情勢の代理 | yfinance |

**追加候補 7 件（Tier 2）**:
- セクター ETF: XLK / XLF / XLE / SMH（半導体）
- 暗号資産: BTC-USD（流動性相場のセンチメント）
- コモディティ: HG=F（Dr. Copper、世界景気）
- 海外指数: ^N225（日経225、米市場クローズ後の補完）

**除外推奨**:
- 既存 `VTI` は SPY と重複 → 削除推奨
- 銀(SLV) / EUR/USD（DXY で代替）/ 上海総合 / DAX / Put-Call比率 / AAII（API 取得困難 or 規約リスク）

**データ精度の懸念と推奨修正**:
- VIX/^TNX は FMP `batch_quotes` で取得失敗ケース多 → yfinance フォールバック必須
- USD/JPY: FMP は `USDJPY`、yfinance は `JPY=X` → ティッカー二系統管理層が必要
- 60秒キャッシュ → **動的 TTL**（通常 60s / 米市場 8:30-10:00 ET と 14:00-16:00 ET は 15s）
- 米市場クローズ後は **「Closed」バッジ**を出し、動いている群（FX/原油/BTC）と停止群を視覚分離
- `change_pct` の基準（前日比 / 24h / settlement）が市場ごとに違う → ツールチップで明示

### 14-2. 表示方法（Webアプリ設計エキスパート視点）

**推奨: タブ切替 + 2行グリッド**（Notion DB view 流）
```
┌─────────────────────────────────────────────────┐
│ [主要] [為替・債券] [セクター ETF] [海外]       │ ← チップタブ
│ ┌────┬────┬────┬────┐                           │
│ │SPX │NDX │DJI │QQQ │  ← 2行グリッド固定        │
│ ├────┼────┼────┼────┤                           │
```

- カテゴリ内のみ横スクロール許容、チップ active 時のみシアン 1px border

**代替: 主要 4 + Disclosure ドロワー**

### 14-4. 時事ニュース セクション設計

**配置**: 検索バー直下、ウォッチリストの**上**
- 理由: マクロニュース = 市況の文脈、個別銘柄判断（ウォッチリスト/Movers）の前に来るべき
- Bloomberg Terminal の "市況サマリー → 個別" 思想

**形式**: 「Today's Brief」型 — 縦リスト 3 件即読み + 「すべて表示 →」で展開
```
┌─────────────────────────────────────────┐
│ Today's Brief             すべて表示 →  │
├─────────────────────────────────────────┤
│ [HIGH] マクロ・地政学    2 時間前 翻訳済│
│ S&P500 年末目標、GS が 6,500→6,800 に  │
│ 上方修正。利下げ織込みと AI 設備投資   │
│ Goldman Sachs · 関連: SPY NVDA →       │
└─────────────────────────────────────────┘
```
- **1 件**: 見出し 16px bold + 要約 60〜80字 (13px, line-height 1.6) + ソース + 経過時間 + 関連ティッカーチップ
- **重要度バッジ**: `HIGH`(amber) / `市場全体`(シアン) / `地政学`(パープル) — 文字+色で色覚配慮
- **左端 2px 縦バー**で impact 視覚化（amber=高 / cyan=中 / gray=低）
- 24h 超 opacity 0.5、48h 超は「過去のニュース」アコーディオン格納
- クリック = **インライン展開**（モーダルは離脱誘発で却下）、外部リンクは矢印明示
- カルーセルやサムネ画像は **入れない**（30秒スキャンを破壊）

**データソース選定**（推奨スタック）:
| ソース | コスト | 用途 |
|---|---|---|
| **FMP General News** (`/v3/general_news`) | 既契約に含む | メイン一次ソース |
| **AlphaVantage NEWS_SENTIMENT** | 無料 25req/日 | sentiment スコア補強 |
| **Claude Haiku** | ~$0.10/日 | L3 LLM 分類 + 日本語翻訳要約 |

**マクロ判定 3 層フィルタ**:
1. **L1 キーワード辞書** (~80語): Fed/FOMC/CPI/PPI/NFP/GDP/Treasury yield/oil/OPEC/Iran/China tariff/S&P 500/recession/IPO/Powell/ECB/BOJ 等

### 15-3. 期間切替 推奨実装方針

**データ層（Webアプリ設計エキスパート視点）**:
- **推奨: ヒストリカル一括取得 + クライアント派生**
- `/api/market-history?period=1Y` で 1Y 日足を全指標分一括取得 → 1D/1W/1M/6M はクライアント側スライス計算
- 期間切替が **完全 0ms**（Bloomberg / TradingView 体感）
- バックエンドキャッシュ: 当日値 60s / 履歴 1h / 1Y 終値 24h
- 容量: 30 銘柄 × 252 営業日 × 8byte ≈ 60KB → Python dict + TTL で十分、Redis 不要
- クライアント: **localStorage + SWR (stale-while-revalidate)** で段階表示

**計算ロジック（金融アナリスト視点）**:
- 取引日基準で統一: 1W=5 / 1M=21 / 6M=126 / 1Y=252 営業日前
- 配当: 価格リターン（adjusted close、スプリットのみ調整）が個人投資家に直感的
- VIX / 金利 / BEI 系は **絶対値変化（pt 表示）**、`display_mode: 'pt'|'pct'` メタフラグで出し分け
- データソース: **yfinance 単一ソースに統一**（FMP との混在で 6M/1Y で数 % ズレるリスク）

**UI（UI/UX デザイナー視点）**:
- **チップタブを右上配置**: pill 型 `[1D] 1W 1M 6M 1Y`、選択 chip のみシアン地・白文字
- chip 高 24px + タップ領域 32px、横 padding 10px、font 11px / 500
- 切替アニメ: **opacity 0→1 (120ms) クロスフェードのみ**（数値カウントアップは禁止 — 「データが動いてる」と誤認）
- 段階フェード: `transition-delay: index * 8ms` で 1 行ずつ更新（最大 240ms 以内）
- `prefers-reduced-motion` で全アニメ無効化

**期間別の色帯閾値（重要）**:
| 期間 | ±閾値 | 理由 |
|---|---|---|
| 1D | 1.0% | 通常レンジ |
| 1W | 3.0% | 週次ボラ |
| 1M | 5.0% | 現状維持 |
| 6M | 15% | セクター ETF 想定 |
| 1Y | 25% | 年次レンジ |

色帯透明度を 1Y 時に 60% → 30% へ自動減衰（全画面緑/赤の防止）。

### 16-3. 期間切替 (1D/1W/1M/6M/1Y) 実装方針

- `/api/market-history?period=1Y` で 30 指標 × 252 日 を **1 リクエスト一括** (≈15KB gz後)

#### E. 計算ロジック (金融アナリスト推奨)

- **デフォルトは平均取得単価** (Robinhood / 楽天証券準拠、日本人直感)
- FIFO は「税務用」副次表示
- **配当再投資**: `lot_type='dividend_reinvest'` で別 lot 記録、自動取得は Phase X-3
- **株式分割**: FMP `/historical-price-full/stock_split` で取得 → lot 自動補正、Cron 日次
- **為替**: USD 建て保持、表示時に当日 JPY 換算、円ベース/ドルベース損益トグル切替
- **計算位置**: クライアント (Zustand selector + useMemo)、リアルタイム株価で再計算

#### F. データ精度の落とし穴 トップ 3

1. **株式分割の遡及調整**: NVDA 10:1 (2024/6) 後の取得単価入力で計算崩壊
2. **為替の扱い**: USD 単一保存 + 表示時換算、為替手数料・スプレッドは含めない
3. **平均取得単価**: 加重平均一択 (FIFO は税務用、UI に出すと混乱)

- `tagsStore`: tags / watchlist_tags

- localStorage 永続化は `persist` middleware で 1 行
- Supabase Realtime subscription を store 側で集約

**ただしタグ X-1 単独先行なら useTags フック単独で OK** (Holdings 着手と同時に Zustand 移行)。
