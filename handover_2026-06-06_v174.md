## 🔴 次セッション最初の作業: count-up「入場時リセット (0 になる動き)」の解消

user 要望「count-up/ゲージが view 内入場時に『最終値 → 0 → 上昇』と一瞬リセットするのが不自然。0 にならないようにしてほしい」(soft 要望「できれば」、移行のため未着手)。

**真因**: count-up 表示が `inView ? animated : FINAL`。view 前は最終値を表示 → view-in で animation が 0 から始まる → 「最終値 → 0 → 上昇」のリセットが見える。**バーは既に `inView ? w% : 0`（0→伸びる）で正しい**ので、numbers/arc も同方式に揃える。

**fix (option 2: view 前から 0、view-in で 0→値に count up、リセット無し)** — 各表示の「before inView」分岐を FINAL → 0 に:
- `frontend/src/components/ForwardOutlookSection.jsx` MetricBlock: `fmt(inView ? animConsensus : consensus, currency)` → `: 0`
- `frontend/src/components/GuidanceCard.jsx`: ①数字 `Math.round(inView ? animatedAchievement : achievementPct)` → `: 0` / ②`<ArcProgress value={inView ? animatedAchievement : achievementPct}` → `: 0` / ③surprise `(inView ? animatedSurprise : surprisePct)` (式中 2 箇所) → `: 0`
- `frontend/src/components/AnalystPanel.jsx`: ①TargetPriceRangeBar legend `fmtUsd(inView ? animatedLow : lowSafe)` (low/median/high) → `: 0` / ②RecommendationStackedBar legend `Buy {animBuy ?? buy}` → `Buy {inView ? (animBuy ?? 0) : 0}` (Hold/Sell 同様)
- → build → commit → deploy → **cold 銘柄で dogfood**（section を scroll-in → 数字/ゲージが 0 から count up、「最終値→0」のリセット無し）。
- ⚠️ tradeoff: option 2 は view 前 (画面外) に 0 を表示する。user が「0 表示も嫌」なら count-up 撤去 (静的) が代替。**user は count-up 維持 + リセット無しを希望**したので option 2 が解。dogfood で最終確認。
- 検証 gotcha: hidden タブは IO/rAF 停止 → 計測前に `document.visibilityState==='visible'` 確認 ([[feedback_chrome_mcp_visibility]])。ただし summary の fetch 駆動状態 (skeleton↔text) は hidden でも観測可。
