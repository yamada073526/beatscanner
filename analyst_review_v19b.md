レビュー日時: 2026-04-29
レビュアー: CFA保有 米国株アナリスト視点

## ✅ 妥当な点

### 1. 指標の組み合わせ設計
EPS・CFPS（一株あたり営業CF）・営業CFマージン・売上高という4軸の組み合わせは、収益性・キャッシュ創出力・成長継続性を同時に評価できる点で合理的である。CFAカリキュラムにおいても「EPSとOCFを並列で確認すること」は基本とされており、アナリストによる一般的なスクリーニング手法と整合する。

### 2. CFPS > EPS の条件（条件⑤）
これはアクルーアルの質（Accrual Quality）を測る代理変数として機能する。会計上の利益（EPS）が営業キャッシュフロー（CFPS）を上回る状態が続く場合、非現金項目への依存・積極的な会計処理（積極的な収益認識、減価償却の圧縮）が示唆される。`judgment.py` での実装 `p_t.cfps > p_t.eps` は直近期のみの比較だが、条件③のCFPS3期連続増加と組み合わせることで実質的に複数期の監視をカバーしている。

### 3. 連続増加の定義
`_is_strictly_increasing_and_positive` 関数は「全期間正値 かつ 単調増加」を要求しており、途中でゼロ・負値があれば即時FAIL。この厳格さは「一時的回復銘柄」を排除する意図として適切である。EPS については最新期のみ正値チェックに緩和（`_is_strictly_increasing_and_last_positive`）しており、赤字転換後のV字回復企業を取り込める設計上の判断も合理的である。

### 4. データソースの冗長化
FMP→Alpha Vantage→yfinance のトリプルフォールバック構成（`_fetch_eps_data` 内）は商用グレードのロバスト性に近い。単一ソース依存のツールに比べ、実績EPS・推定EPSの欠損率が大幅に低い。また FMP Limit Reach 時に自動的に次ソースへ切り替える実装は、個人投資家向けの無料プランに対する現実的な対処として評価できる。

### 5. ガイダンス取得アーキテクチャ
SEC EDGAR 8-K（Items: 2.02）から直接 EX-99.1 を取得し、Claude API で要約するパイプラインは高品質である。公式一次情報に近いソースを使用しており、ガイダンス情報の信頼性は高い。Motley Fool トランスクリプトをセカンダリとして使用するフォールバックも実用的である。

### 6. 可視化プロンプトの財務的正確性
`visualizer/prompt.py` の RULES は多くの点で適切に設計されている。特に：
- `operatingMargins` を `operating income / revenue` と定義していること
- Beat/Miss 判定の直近期限定適用（past periods は estimate: null）
- バリュエーション指標（P/E・P/B・P/S）の業界平均比較を「割安/適正/割高」で分類する設計

## ⚠️ 要改善（データ・ロジック）

### 1. AI要約②行目の数値誤り（最重大）

スクリーンショットの②行目に次の記述がある：

> EPS $13.64（予想$3.92に対し実績$4.14でBeat、サプライズ+5.6%）、売上高$281.72B（前年比+14.9%）、営業キャッシュフロー**-$136.16B**を達成し前善

以下の3点で重大な誤りが確認できる：

**問題 A: EPSの二重表示・矛盾**
冒頭「EPS $13.64」とありながら、括弧内で「予想$3.92に対し実績$4.14」としている。これは四半期EPS（実績$4.14）と年次EPS（$13.64）が混在している可能性が高い。Q3 FY2025 のMSFT EPS実績は約$3.30〜$3.46（2025年10月時点）前後であり、$13.64はFY2024年次EPSに近い。判定ロジック（`judgment.py`）は**年次データ**のみで5条件を評価するが、AI要約プロンプトには四半期 Beat/Miss データ（`beat_miss_detail`）も渡している。LLMが年次EPS（$13.64）とQOQ Beat情報（$4.14）を混在表示した結果と推測される。

**問題 B: 営業CFの符号**
「営業キャッシュフロー-$136.16B」とあるが、MSFTの営業CFは正値（FY2024で約+$118B）である。負符号は明らかなエラーであり、LLMが支出CF（投資活動や財務活動）を誤って参照したか、プロンプトに渡った数値のスケール変換エラーの可能性がある。

**問題 C: 売上高の「前善」**
「前善」はおそらく「前年比」の誤字または文章未完了状態であり、生成テキストの品質管理不足を示している。

**根本原因**: `build_user_prompt` 関数が `metrics_trend`（年次3期分）と `beat_miss_detail`（四半期実績）を同一プロンプトに渡しているが、LLMに対してどちらのコンテキストで各数値を使うべきかを明示する制約がない。

### 2. 営業CFマージンの期間ミスマッチリスク

`judgment.py` の条件①は「直近年次期（T）の営業CFマージン ≥ 15%」のみを評価する。**直近の四半期決算が弱い場合でも、年次CFマージンが15%を超えていれば条件PASSとなる**。例えばMSFTがある四半期に一時的コスト増でCFマージンが12%に落ちても、年次集計では18%程度を維持し条件をPASSする。投資家が四半期決算で判断したいシーンに対し、このラグは判定の遅行性を生む。

### 3. EPS連続増加の定義：会計EPS vs 調整EPS

`judgment.py` は `epsDiluted`（GAAP希薄化後EPS）を使用している。米国の大型テック企業（META・GOOGL・AMZN等）は株式報酬費用・減損・訴訟費用を除いた **Non-GAAP EPS（Adjusted EPS）** を決算発表の主軸とするケースが多い。GAAP EPSは法律上正確だが、アナリストの consensusや経営陣の見解との乖離が大きい場合がある。例えば AmazonのGAAP EPSは投資損益で大きく振れる。この乖離が CFPS > EPS 判定（条件⑤）のロジックとも干渉する可能性がある。

### 4. CFPSの計算基準

`cfps = operating_cf / shares_diluted` として計算しているが、一般的な財務分析では Operating CF per Share は **基本株式数（basic shares）** で割るケースも多い。希薄化後株式数は将来的な株式報酬・CB転換を含むため、相対的にCFPSが保守的になる。これ自体は安全サイドの設定であるが、他のデータソースとの比較時に不整合が生じる可能性がある。また、株式数データが欠落（`shares_diluted = 0`）の際に CFPS = None となり条件③がFAILになるケースがある（`_is_strictly_increasing_and_positive` は None があれば False を返す）。株式数の別フィールドへのフォールバックを強化すべきである。

### 5. 判定スコープ：年次データのみ

5条件の判定は**年次（annual）データのみ**で実施している（`period="annual"` 固定）。米国株分析において、決算サプライズの判断軸は四半期（QoQ・YoY）であることがほとんどである。年次で3期連続増加していても、直近2四半期が連続減速していれば「ピークアウト懸念」が台頭する。現状の設計では、成長の鈍化・減速が判定に反映されるまでに最大1年のラグが生じる。

### 6. Beat/Miss 閾値（±3%）の合理性

`_verdict` 関数の Beat/Miss 閾値は ±3%（サプライズ率）であるが、業界・時価総額・レビュアーによって適用される閾値は異なる。小型株やバイオ系では ±1% でも株価が大きく動くことがある。また、EPS予想が$0.01未満（ほぼゼロ付近）の場合は % 計算が発散するため、`abs(estimated) < 0.05` で絶対値差分に切り替える実装は適切だが、その旨の表示上の注記がUIに存在しない。

## ❌ 追加すべき指標・視点

### 1. 売上高成長の加速・減速トレンド（Revenue Growth Acceleration）

現状は「売上高が3期連続増加しているか否か」の二値判定のみ。プロの分析では成長率そのものの変化（加速か減速か）が重要である。例えば $200B → $220B → $240B（均一10%成長）と $200B → $240B → $246B（減速型）は両方PASSになるが、後者は「ピーク感」が強い。**YoY売上高成長率の前期比変化（growth acceleration ratio）** をサブ指標として追加すべきである。

### 2. 粗利益率（Gross Margin）と営業レバレッジ

営業CFマージンは資本効率の包括指標だが、**粗利益率（Gross Margin）の推移**は事業構造の変化を最も早期に映す。MicrosoftのAzure比率拡大・Apple のサービス比率上昇など、高マージンセグメントへのミックスシフトはGross Marginに先行して現れる。CFマージンとGross Marginの乖離（オペレーティング・レバレッジ）も重要な判断材料である。

### 3. 残余フリーキャッシュフロー（FCF = OCF - CapEx）

営業キャッシュフロー（OCF）は設備投資（CapEx）を含む「入ってきた現金」であり、株主価値に最も直結するのは **FCF（フリーキャッシュフロー）= OCF - CapEx** である。MSFTはデータセンター拡張でCapExが急増しており（FY2024で約$44B）、OCFが高くてもFCFマージンは相当低下している。「キャッシュ創出力が堅牢」という評価はFCFベースで検証されるべきであり、OCFのみに依拠した評価は過大評価につながりうる。

### 4. 株主還元の持続可能性（Payout Sustainability）

可視化プロンプトには `dividend.payoutRatio` および `dividend.buyback` フィールドが存在するが、これらはAI生成（Claude推論）であり、実際のFMP/yfinanceデータからの計算ではない。配当性向・自社株買い規模の時系列推移をFCFとの比較で表示することで、「還元が持続可能か過剰か」の判断が加わる。

### 5. バリュエーション：PEG レシオ

P/E・P/B・P/S は静的なバリュエーション指標であり、成長株の評価には不向きな側面がある。**PEG比率（P/E ÷ EPS成長率）** は成長を織り込んだ相対バリュエーションとして広く使われており、「高P/Eだが高成長でPEG < 1なら割安」という判断が可能になる。現状の可視化プロンプトにはPEGフィールドが存在しない。

### 6. 機関投資家動向（Institutional Ownership の変化）

広瀬氏の投資哲学第3条「業界No.1企業への投資を先ず検討しろ」と連動して、**機関投資家の持ち株比率変化（13F動向）** を表示することはシグナル強化につながる。大型機関のポジション積み増しはファンダメンタルズを超えた需給要因として機能する。

### 7. セクター比較（Sector Relative Performance）

5条件は絶対値基準（営業CFマージン ≥ 15%、等）であるが、産業別のベンチマークとの相対評価がない。例えばユーティリティ（電力・ガス）や製造業では営業CFマージン15%は高ハードルである一方、SaaSセクターでは20%以下が物足りないと見られる場合もある。セクター中央値に対する超過率（例：セクター平均より+X%pt）を追加表示することで、業界横断的な比較優位が可視化される。

### 8. 為替感応度（非米国株・ADR対応）

`judgment.py` は JPY・KRW・CNY・HKD の通貨スケールに対応しているが、ADR（米国預託証券）として上場する非米国株のEPS・売上をドル換算する際の為替変動リスクについて注記がない。円安局面での日本株ADR分析では、ドル換算後の数値が実態より良好に見える場合がある。

## ガイダンス判定の是非

### 現状の実装

ガイダンス「非開示」は UI 表示上 NEUTRAL として扱われている。`_fetch_sec_guidance_cached` が `None` を返した場合（ガイダンスなし・取得失敗いずれも含む）、フロントエンドでは「ガイダンス：非開示」と表示し、5条件の判定には影響しない設計である。

### 妥当性の評価：条件付き妥当

**妥当と言える理由**：
1. Apple（AAPL）のように「数値ガイダンス非開示」が意図的な企業方針である場合は多い。これをFAILにすると、業界最高クラスの収益性を持つ銘柄が常にFAILになる不合理が生じる。
2. ガイダンス開示率は業種・文化圏によって大きく異なる（日本株ADRは特に非開示が多い）。

**問題のある点**：
1. **「非開示」と「取得失敗」の区別がない**: 現状の実装では、SEC EDGAR fetch エラー・Motley Fool timeout・データ未整備のいずれも `None` となる。これらはユーザーに区別なく「非開示」と見える。実際には「データ取得を試みたが失敗した」ケースをNEUTRALと扱うことは、情報の透明性を欠く。

2. **ガイダンス減額をNEUTRALにするリスク**: 企業が次期ガイダンスを大幅に引き下げた（guidance cut）場合でも、5条件判定への影響はゼロである。実際の投資判断では「ガイダンス達成・未達」よりも「ガイダンスの方向性（上方修正 vs 下方修正）」の方が株価反応が大きいケースが多い。

3. **前期ガイダンス達成率の未追跡**: 「会社が以前のガイダンスを達成したか」という履歴がない。過去3期のガイダンス達成率を条件に加えることは、経営の実行力（execution quality）を評価する有効な指標となる。

**推奨**: 現状のNEUTRAL扱いを維持しつつ、以下の3段階で段階的に精緻化する：
- `guidance_status: "not_disclosed_by_policy"` / `"fetch_failed"` / `"available"` を区別して返す
- ガイダンス取得成功時は「上方修正 / 維持 / 下方修正」の方向性を判定サブスコアとして追加
- 前期ガイダンス達成率（直近2件）を表示

## 総合評価

beatscanner は「じっちゃまプロトコル」という明確な投資哲学をコードに忠実に実装しており、**個人投資家向けの銘柄スクリーニングツールとして基礎品質は十分に高い**。特にデータソースの冗長化・CFマージン基準・CFPS > EPS のキャッシュ品質チェックは、簡易ツールでは省略されがちな財務的に重要な点をカバーしている。

一方で、プロのアナリスト観点からは以下の3点が最優先の改善対象となる：

1. **AI要約の数値混在バグ**（年次 vs 四半期EPSの混在）はユーザーに誤った投資判断を促す可能性があり、即時対応を要する。
2. **FCFの欠如**は特にCapEx集約型の成長企業（半導体・クラウド）評価で誤判定を生むリスクがある。
3. **ガイダンス情報の不透明性**（非開示とフェッチ失敗の区別なし）はユーザーの信頼性評価を誤らせる。

これらの改善が行われれば、機関投資家の一次スクリーニングツールに近い品質に到達できるポテンシャルを持つ。
