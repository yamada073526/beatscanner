## 3. 触ると危険な箇所 (v102 + v103 教訓)

### v102 から継承
- sticky 検索バー / 発光系 / VITE_ARG/ENV / Hallucination Guard / FMP /stable
- ReadingMode 全面 Overlay (旧 PanelGroup 廃止、 close→list は Linear/Gmail 流)
- NewsItem `ws-pane4-news-item` CSS は v66 §1 round 3 確立済 backwards 戦略
- NewsItem 1 行 line-clamp (タイトル切れる場合 ReadingMode で全文確認、 dogfood criticism 来たら 2 行 + fontSize 11px に pivot)

### v103 新規教訓
- **Pane4Inspector orchestrator 化後の hook 順序**: useNewsFeeds → useSignalPipeline → useTranslation → usePrefetchTopNews が dependency chain。 hook の引数を入れ替えると derived data が壊れる
- **store hoisted state (selected / pane4Section) は hook の外、 Pane4Inspector の直接管理**。 hook 内で `useWorkspaceStore` を呼ぶと subscriber が増えて re-render 増 (Frontend Architect verdict)
- **hook 抽出時の API import 整理**: 旧 Pane4Inspector の `fetchMacroNews / fetchNewsBulk / translateTexts / buildSignals` import は全て hook 側に移動。 orchestrator は API 直接 import 禁止 (single responsibility 維持)
- **MacroLensPanel / ReadingRoomPanel は最小限の wrapper**: 各 5-50 行、 1 個の component 内で複数の責務を持たない (test 容易性 + onUpgrade 等の追加 prop 拡張容易)
