## 🟡 重要知見 (永続化候補・特にデータ壁は数ヶ月有効)

- **ガイダンス beat 連続 (8Q streak) は構造的データ壁で当面不可能** (Sprint 4c DEFER 確定の根拠):
  - `guidance_snapshots` 全 **69行**。8Q に近いのは SNOW(6Q)/AMZN(7Q)/NVDA(6Q) のみ、かつ filed_at が全て **2026-06-06 以前** → PIT consensus (蓄積開始 2026-06-06) が 1 件も無い。
  - PIT 適格な filed_at を持つ銘柄 (FDX/BB/JBL 等) は guidance が 1〜2Q のみ → streak 不能。
  - backfill でも consensus 履歴自体が遡れず過去の vs-consensus は再構築不能。
  - **数ヶ月後に nightly が新規8-Kに filed_at 付与 + consensus 履歴深化で自然に可能化**。再着手はその時。
- **5条件 all_passed は実データで極めて稀** (最新 198銘柄中 **1**、全体 0.74%)。真因は capex でなく **②③④の「3期連続増加」monotonic 要件** (実 pass率 ①66 ②10 ③9 ④23 ⑤73%)。→ 二値 streak は不採用、passed_count 単体も誤誘導のため per-condition ヒートマップを主軸にした。
- **新 endpoint `GET /api/earnings-evaluation/{ticker}`** (main.py 末尾・6h cache・RLS public read・auth 不要・@no-llm/§38-safe)。frontend は `fetchEarningsEvaluation` (dedupGet)。`earnings_evaluation` は anon SELECT 可 (`earnings_evaluation_public_read`)。
- **純関数 `deriveEvalContinuity`** (L3QualityFold.jsx) に回帰テスト 8 件 (古→新整列 / 安定 rate≥0.75・ネック≤0.25 かつ ≥4Q / null 安全 / latest)。
- A6「過去ピーク N/5」pillbox は今回未採用 (将来 enhancement 余地・金融 reviewer 推奨)。
