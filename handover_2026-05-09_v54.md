### 主要バグの解明 (3 体合意)

**根本原因 (★★★★★)**: v53 で `.surface-card { contain: layout paint }` を追加したが、`paint` 値は **要素の bounding box 外に descendant が描画されるのを clip する** 仕様。box-shadow は要素の outside に描く decoration なので、`contain: paint` 下では **shadow 自体が clipping mask で消滅**。その結果ホーム画面で発光が「乗ったり乗らなかったり」していた。

**副因 (★★★★)**: v53 で追加した `:has(.surface-card.is-arriving)` 親抑制ルールが、**子に `.surface-card` がある親 `.panel-card`** を意図せず黙らせていた (e.g., Portfolio 内に Chart があれば Portfolio Hero が抑制される)。

→ v54 で両方撤去 + leaf 優先を JS 側 (`visible.contains(other)`) で 1 箇所に集約。

## 3. 🟡 重要知見 (v54 セッション固有)

### 3-1. `contain: paint` は box-shadow を要素境界でクリップする
`contain: paint` は CSS Containment Level 1 仕様で「**要素の bounding box 外に descendant が描画されるのを clip する**」(MDN 明記)。box-shadow は outside decoration なので **shadow 自体が消滅**する。
→ glow 系の演出を持つ要素には `contain: layout style` までに留める。`paint` 値は overflow:hidden + isolation:isolate で代替可能。

### 3-2. CSS `:has()` での親子発光抑制は副作用が大きい
`:has(.surface-card.is-arriving)` は意図したコンポーネントだけでなく、**任意のネスト構造で子に対象クラスがあれば全部親を黙らせる**。命名規約が完全でない場合 (= 現状の beatscanner) は CSS で書くより JS 側で `visible.contains(other)` で leaf 判定するほうが安全。

### 3-3. hysteresis ロジックは `lastDistance` でなく **active vs candidate の距離差** で組む
v53 の hysteresis は `Math.abs(bestDist - lastDistance) < 16` (= 前 frame の bestDist との差) で組まれていたが、これは「**スクロール速度依存**」で正しく動かない。実際にチラつきが起きる場合は `(activeDist - bestDist) > 24px` (= active より 24px 以上近い候補にだけ切替) にすべき。v54 では hysteresis 自体を削除して観測。

### 3-4. `.is-holding:hover` の cyan inset は gold アクセントと衝突
ウォッチリストで保有銘柄は左 2px の gold バー (`::before`) で識別している。`.ticker-row-v2:hover` の cyan inset 3px shadow と重なると視覚汚染。
→ `.is-holding:hover` だけ inset shadow を **outer shadow + gold tint background** に置換。

### 3-5. analyze CTA は中央寄せ + モバイル全幅
業界慣習 (Stripe / Linear / Notion / Wealthfront) は行内 CTA = 左寄せだが、ユーザー希望優先で中央寄せ採用。モバイル (≤640px) では width: 100% で拇指リーチ最大化 (Apple iOS Sheet 流)。

### 3-6. dev-only debug 属性は import.meta.env.DEV で tree-shake される
`useArrivalSpotlight` で `data-spotlight-active="1"` / `data-spotlight-debug="1"` を DEV 時のみ付与 → CSS で magenta outline で可視化。本番では `import.meta.env.DEV === false` で属性付与コードが除去される (Vite tree-shaking)。
