## 5. 🟡 重要知見 (v56-v60 セッション固有)

### 5-1. 発光バグの根本原因 (v54-v59 で完全解明)
- `contain: paint` が box-shadow を要素境界でクリップ (v54)
- `:has(.surface-card.is-arriving)` 親抑制が意図せぬ親を黙らせていた (v54)
- HomeTab L156 の inline `border` が `.is-arriving` の border-color に勝つ → `!important` で対応 (v55)
- `.bs-panel:hover` の dark mode 専用ルール欠如 → 追加 (v57)
- `.is-arriving` (specificity 0,2,0) > `:hover` (0,1,1) で arrival 中 hover 発火しない → 複合 selector で解決 (v57)
- `.bs-panel` 背景 37% 暗い → `.panel-card.is-arriving` の dark を ring 1.5px に強化 (v57)
- ChartTab root の透過 + 内部 wl-list-frame の border で二重枠 → `surface-card` を `wl-list-frame` に移動 (v59)

### 5-2. 7 体エージェント全員一致 (v60 plan 承認の根拠)
- **ホーム = 1 pane 維持** が正解 (Robinhood/Apple/Yahoo/Bloomberg 全社採用)
- **6 タブ + 3 pane 構成** (ホーム/判定/スクリーナー/決算/記事/学習)
- **Pro ¥980 + Premium ¥1,800 の 2 段階** が日本人 SaaS 心理閾値 (¥2,000 の壁直下) と業界中央値で最適
- **Next.js 全面移行は今すべきでない** (Web 開発/Web 設計/Anthropic 全員一致、ROI 悪、ペイン構成完了後の 7-8 月以降に再評価)
- **shadcn/ui Hybrid 採用** (新規機能のみ、既存独自 CSS は永続保護)

### 5-3. CLAUDE.md ルール「じっちゃま」の hook ブロック
ユーザー側 hook で UI 文字列に「じっちゃま」を含むコードはブロックされる。コメント内でも grep ベース検出のためブロック。
→ 代替: 「ファンダメンタル 5 条件」「独自プロトコル」を使う。

### 5-4. planGating の責務分離 (v60 設計の核)
- `PLAN` enum (free / pro / premium)
- `FEATURE_GATES` table で機能と必要プランをマップ (30+ 機能)
- `getPlan(subscription, byokKey)` で subscriptions テーブル + BYOK fallback
- `canUse(feature, plan)` で機能ゲート (rank 比較)
- `requiredPlan(feature)` で UI に「Pro/Premium で解放」を出す時に使用

### 5-5. backend price ID resolver の fallback chain
- 旧 `STRIPE_MONTHLY_PRICE_ID` → 新 `STRIPE_PRO_MONTHLY_PRICE_ID` への漸進移行
- `_STRIPE_PRICE_ENV` table で (tier, plan) → env name list を 1 箇所管理
- 既存 BYOK ユーザーの subscription は `_resolve_tier_plan_from_price_id` で 'pro' に逆引き
