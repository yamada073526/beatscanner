### 発光バグの追加真因 (v54 で contain: paint 撤去後も残った問題)

**HomeTab.jsx L155-156** で MoversCard wrapper が `<div className="panel-card rounded-2xl shadow-sm" style={{ background: 'var(--bg-card)', border: '1px solid var(--border)' }}>` のように **inline `border` を直書き**していた。inline style は specificity が最強 (1,0,0,0) なので、CSS 側の `.panel-card.is-arriving { border-color: ... }` (specificity 0,2,0) は **絶対に勝てない**。

## 3. 🟡 重要知見 (v55 セッション固有)

### 3-1. inline style は CSS specificity の最強敵
`<div style={{border: '...'}}>` は specificity (1,0,0,0) で **CSS のあらゆる class ルールに勝つ**。CSS で `.is-arriving { border-color: ... }` を書いても適用されない。
→ 解決策 1: CSS で `!important` を付ける (緊急対応、v55 採用)
→ 解決策 2: inline style を CSS class に移行 (恒久対応、§2-2 #5 で実施推奨)

### 3-2. `content-visibility: auto` は CLS=0 維持に `contain-intrinsic-size` 必須
`content-visibility: auto` で画面外要素の paint/layout を skip できるが、要素高さの reserve がないと **scroll bar 位置がジャンプ** = CLS。`contain-intrinsic-size: 0 360px` (横幅は親に合わせ、高さは 360px と仮定) で初期高さを reserve。

### 3-3. focus-visible は Tab キーでのみ発火、マウスクリックは無視
WCAG 2.4.7 / Vercel Geist UI 流。kbd nav 中だけリング表示で、マウスクリック時は表示しない (`:focus` だと両方発火する)。
→ a11y AAA を満たしつつ、マウス操作を邪魔しない。

### 3-4. data-flash 属性駆動で React minimal change
React 側で `data-flash="up"` を付与すれば CSS animation が走る。animation 終了で removeAttribute。これにより:
- React 側のロジック変更最小 (前回値 ref + setAttribute だけ)
- アニメーションは CSS で完結 (GPU 化、prefers-reduced-motion 自動無効)
- 再利用可能 (`.bs-stat-hero` `.bs-stat-secondary-value` だけでなく将来の `.pd-num` 等にも適用可)

## 4. 触ると危険な箇所 (v47 〜 v55 累積)

### v55 追加
- **inline style に勝つには `!important` か CSS class 移行** (§3-1): `.is-arriving { border-color: ... !important }` パターンは緊急対応で恒久解決ではない
- **`content-visibility: auto` は `contain-intrinsic-size` セット必須** (§3-2): 単独使用で CLS 悪化
- **focus-visible は `:focus` と違う** (§3-3): マウスクリックでは発火しない、kbd nav 専用
- **`data-flash` 属性駆動** (§3-4): React で setAttribute、animation end で removeAttribute、CSS で animation/keyframes 定義
