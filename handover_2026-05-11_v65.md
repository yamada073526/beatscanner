### §4-A round 25: Pane 3 bs-panel rest shadow

3 体レビュー結論 (engineer/UX/marketer 合議) で「二重カードシステム」が「なんとなくダサい」の root cause と判明:
- `.bs-panel ds-card` (Hero / KpiStrip / VerdictDetail / ConditionGrid / Profile / EPS 推移): 透明 bg + 1px slate border、**rest shadow なし**
- `.panel-card` (HistoryChart / InsightsPanel / StockPriceChart / NewsPanel / IRLinksPanel): filled bg `rgb(30,36,51)` + `rgba(0,0,0,0.5) 0px 1px 3px` shadow

修正:
```css
.ds-judgment-detail .bs-panel:not(:hover):not(.is-arriving) {
  box-shadow: var(--shadow-1);
}
```
- Pane 3 scoped (他 page の bs-panel に波及せず)
- `:not(:hover):not(.is-arriving)` で発光バグ系 compound 4 セット規律と非衝突
- Token `var(--shadow-1)` 使用 (ALLOWED-SHADOW token 経由、raw shadow ではない)
- 検証: bs-panel rest shadow = `0 1px 0 rgba(0,0,0,0.3), 0 1px 2px rgba(0,0,0,0.2)` (dark)

却下された H1 仮説: `contain: layout` で box-shadow clip — resting cards に shadow なしのため clip 対象なし

## 2. v65 後半: dogfood で発見した 3 バグ修正

### Bug 1: kebab dropdown が下のペインに隠れる
- **原因**: `<header class="ds-ws-header">` (56px) と `<div class="ds-workspace-shell">` 両方が `overflow: hidden`。kebab dropdown が `position: absolute` でも親の overflow にクリップされる
- **修正**: `WorkspaceHeader.jsx` で **`createPortal(menu, document.body)`** + **`position: fixed`** に変更。`useLayoutEffect` で kebab button の `getBoundingClientRect()` から viewport coord (top: rect.bottom+6 / right: window.innerWidth-rect.right) を計算。z-index 9999
- **検証**: 本番で `parentIsBody: true`、menu bottom=78px (header 56 を超える)、`fitsViewport: true`

### Bug 2: Pane 4 toggle 開 → 閉で画面真っ白
- **原因**: `react-resizable-panels` の `PanelGroup` で Panel 数が動的に変動すると `Error: Panel data not found for index 3` で React tree 全 unmount → 白画面。autoSaveId 切替だけでは internal index 同期が間に合わない
- **修正**: `WorkspaceShell.jsx` の `<PanelGroup>` に **`key={pane4Visible ? 'wsg-4' : 'wsg-3'}`** を追加。pane4 visibility 切替時に PanelGroup を明示的に remount し internal index 整合を保つ
- **検証**: 4 サイクル連打 (open→close→open→close) で root child count 1 維持、Panel error ゼロ
- **副作用**: 切替時に Pane 1/2/3 のサイズ調整がリセット (autoSaveId 違いで保存 layout も別なので実害ほぼなし)

### Bug 3: sparkline が 1D 選択時に直線
- **原因**: backend `/api/price-history` が period=1d を無視して 1Y daily (250 点) 返却 + frontend `RowSparkline` が `PERIOD_DAYS['1d']=2` で末尾 2 日 slice → 2 点 polyline → 直線
- **修正**:
  - backend: 新 endpoint **`/api/price-intraday/{ticker}`** (yfinance 5 分足、`period='5d', interval='5m'`、frontend で最新営業日のみ抽出、60s cache)。`backend/app/yfinance_source.py` に `fetch_price_intraday` 追加
  - frontend: `api.js` に `fetchPriceIntraday(ticker)` 追加、`RowSparkline.jsx` の `getOrFetch` で period='1d' は intraday endpoint へ route、`RowSparkline` component は intraday data を slice せず全描画
- **検証**: 全 sparkline polyline pointCount 50-90 (前回 2)、座標が自然なジグザグ pattern。`/api/price-intraday/AAPL` で 78 点取得 (NYSE 9:30-16:00 ET、5 分足)
- **既知制限**: NYSE 休場時 (週末・holidays・session 外) は前営業日 data。低出来高 symbol (^VIX 等) は 50 点に短縮されることあり。yfinance Railway IP cold start で初回 fetch が空配列 → 60s 後の再 fetch で成功
