#### C-1. Cup chip tooltip 改行 (即修正、 1 行)

#### C-4. 旧 UI cutover (重要、 root cause 特定 + 3 段 fix)

**症状**: `?layout=workspace&ticker=MSFT` → backend が `_ogp_html()` 経由で `<script>location.replace("/?ticker=MSFT&__r=1")</script>` を返却 → SPA に redirect されるが **`layout=workspace` が落ちて旧 UI が開く**。 F5 でも同じ。

**Root cause** (backend `main.py:10059`): `/?ticker=XXX` で OGP HTML 返却するが、 redirect URL に query params を保持していない。

**修正 3 段**:
1. backend `root()` に `layout` Query param 追加、 `if not eff or __r or layout: SPA index.html` で OGP スキップ → `?layout=workspace&ticker=MSFT` は OGP 経由せず直接 SPA index.html を返す
2. frontend `App.jsx`: `useWorkspaceLayout = !isMobileForWorkspace && !urlWantsClassic` に変更 (PC default workspace、 `?layout=classic` 明示時のみ旧 UI)
3. mobile redirect ロジック (existing) は維持 (mobile + `?layout=workspace` で SPA fallback)

**影響**: 既存 bookmark で旧 UI を見ていた user は新 UI に cutover される (user 要望直結 + マーケター agent 過去 verdict「mobile は SPA、 PC は workspace」 と整合)。 旧 UI を強制したい場合は `?layout=classic` を URL に追加。

## 3. 触ると危険な箇所 (v77 新規 + 継承)

### v77 新規確定原則

**ProTeaser blur 戦略 (Session 3 暫定実装、 `feedback_technical_signal_thresholds.md`)**:
- frontend localStorage `bs_pro=1` で gate、 backend は send-all で demo flow 維持
- Cup-Handle のみ blur (DMA/RS は Free 表示で送客 = 「価値を見せて Pro へ」)
- blur は CSS filter ではなく Recharts prop (`strokeWidth=3 strokeDasharray="4 4" opacity=0.3`) で曖昧化、 4 層防御と衝突しない
- CTA overlay は `pointer-events: none`、 Recharts hover を殺さない
- Session 4 で Stripe + Supabase profile.tier に置換予定、 localStorage は削除

**RS ranking_label の閾値 (v77 dogfood 教訓)**:
- percentile ≥ 75 → 「上位 X%」 / ≤ 25 → 「下位 X%」 / 26-74 → 「中位」
- 初回実装で `f"上位 {100 - percentile}%"` 一括計算したら「percentile 19 → 上位 81%」 と弱い銘柄を強い表記する致命バグ
- chip tone と整合 (上位 = gain green / 下位 = loss red / 中位 = muted)

**DMA cross は golden のみ (Session 3 確定、 Phase 2 で dead cross 再評価)**:
- じっちゃま buy-only 規律 + Cup-Handle 同様の Phase 3 まで売りシグナル禁止
- dead cross は Phase 2 で portfolio stop-loss 連携時に別 chip (amber) として再導入候補
- `_detect_dma_cross()` の `kind` field は将来 "dead" を追加できる extensible design

**snap harness 90s timeout の規律逸脱記録**:
- visual_harness_exception 60s 推奨だが、 Free/Pro 2 pass + backend cold start で 90s 必要
- 規律違反ではなく「複数 pass 必要時の例外」 として handover に明示記録
- 次セッションで Pane 3 mount 高速化 or harness 構造見直しで 60s に戻すこと
