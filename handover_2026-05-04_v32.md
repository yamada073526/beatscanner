## 0. v32 の概要

v31 で Stripe 決済は実装済みだったが、**入口の動線が壊れていてフローが起動しない状態**だった。Webマーケター視点のレビューで以下4点を特定し、本セッションで修正:

1. **🚨 P0**: InsightsPanel の Pro CTA が `alert()` のままで Stripe フロー未接続 → **完全に機能しない状態**だった
2. **🔥 P1-A**: 未ログイン時「サインインして見る」ボタンが `window.scrollTo(top)` のみで何も起きない
3. **🔥 P1-B**: 未ログイン時に何もチラ見せされず、価値訴求がゼロ
4. **🟢 P3-2**: UpgradeModal で「7日間無料」が価格と並列扱いで埋もれていた

修正後は **登録動線・課金動線が一貫して機能する状態** に。

## 1. v31 → v32 で完了した修正

### 1.1 Fix 1 (P0): handleUpgrade を Stripe フローに接続

**問題箇所**: [InsightsPanel.jsx](frontend/src/components/InsightsPanel.jsx)
```javascript
// 旧（v31 まで）
const handleUpgrade = () => {
  alert("Pro プランは近日公開予定です！");
};
```

Stripe 決済が v31 で実装済みにもかかわらず、TeaserView の「Pro プランを見る →」ボタンを押しても上記アラートが出るだけ。**Pro 加入フローが本番で1件も成立しない致命的バグ**だった。

**修正内容**:
- `InsightsPanel` に `onUpgradeClick` prop を追加
- `handleUpgrade = onUpgradeClick || (() => {})` に置換
- App.jsx 側から `onUpgradeClick={() => upgrade.open('市場の声')}` を渡す → 既存の `UpgradeModal` フローへ正しく接続

**効果**: 「Pro プランを見る」クリックで UpgradeModal → 「今すぐ無料で試す」ボタン → Stripe Checkout の正常フローが復活。

---

### 1.2 Fix 2 (P1-A): 「サインインして見る」→ 即 Google ログイン

**問題箇所**: [InsightsPanel.jsx](frontend/src/components/InsightsPanel.jsx) の未ログインビュー
```javascript
// 旧
onClick={() => window.scrollTo({ top: 0, behavior: "smooth" })}
```

ボタンを押すとページトップへスクロールするだけ。**ユーザーから見ると「ボタンが壊れている」**ように映り、即離脱の原因。Webマーケティング業界で言う "Dead click" の典型例。

**修正内容**:
- `InsightsPanel` に `onSignIn` prop を追加
- App.jsx から `onSignIn={signInWithGoogle}` を渡す
- ボタンクリックで `signInWithGoogle()` が直接発火 → Google OAuth 画面へ遷移

---

### 1.3 Fix 3 (P1-B): 未ログインチラ見せビュー実装

**問題箇所**: [InsightsPanel.jsx](frontend/src/components/InsightsPanel.jsx) の `if (!user)` 分岐
```jsx
// 旧
🔒
会員限定コンテンツ
この銘柄に関する市場の声はログイン後にご覧いただけます
[サインインして見る]
```

何が見られるのか分からず、登録動機が弱い。**Information Gap Theory（少しだけ見せて、続きを欲しがらせる）が働いていない**状態。

**修正内容**: `NonLoggedTeaserView` コンポーネントを新規追加。3 段構造で構成:

1. **ヘッダー**: 📊 市場の声 + ? + **センチメントバッジ（強気/弱気/中立）を結論として公開**
2. **チラ見せ本文**:
   - 上半分: `summary` の冒頭 80 文字を平文表示
   - 下半分: 続き 140 文字を `filter: blur(4px)` + `mask-image: linear-gradient` で**自然にフェードアウト**
3. **CTA**:
   - 「✅ ログインすると見られる内容」3 項目訴求（毎朝更新／構造化分析／キー指標）
   - **Google アイコン入り「Googleで続ける（30秒・無料）」シアン塗り CTA**（`box-shadow: 0 0 12px rgba(34,211,238,0.30)` でグロー）

**バックエンド変更不要**: `/api/insights/{ticker}` は元々認証不要のため、`useEffect` の認証ガード `if (!ticker || !user) return` を `if (!ticker) return` に変更するだけで未ログインでもデータ取得可能。Supabase に既にキャッシュされていればリアルタイム表示。

#### 🆕 デザイン詳細

| 要素 | スタイル |
|---|---|
| 平文プレビュー | padding 14px / `rgba(34,211,238,0.07)` 背景 / `rgba(34,211,238,0.25)` 1px ボーダー / 角丸上のみ |
| ぼかし続き | `filter: blur(4px)` + `mask-image: linear-gradient(to bottom, black 0%, transparent 95%)` / `pointer-events:none` でクリック不可 |
| Google CTA | 背景 `#22d3ee` / 文字色 `#0f172a` / `box-shadow: 0 0 12px rgba(34,211,238,0.30)` / hover で `#06b6d4` |
| Google アイコン | 4色の SVG（黒塗りに変更） |

## 3. 動作確認チェックリスト

### Webマーケター視点での想定ユーザーフロー

#### A. 完全な未ログインユーザー
1. ホームから NVDA 等を分析
2. 「📊 市場の声」セクションへ自動スクロール
3. **センチメントバッジ（例: 中立）を視認** → 「結論だけ見えて続きが気になる」状態
4. summary 80文字を読む → 「続きが ぼかしで見える」 → Information Gap 発動
5. 「Googleで続ける（30秒・無料）」クリック → **即 Google OAuth 画面へ**
6. ログイン完了 → ページに戻る → InsightsPanel が自動的にフルビューに切り替わる

#### B. ログイン済 Free ユーザー（FMP キーなし、Stripe サブスクなし）
1. 銘柄分析後、「📊 市場の声」で **TeaserView (100文字 + ぼかし + Pro CTA)** を見る
2. 「🔓 続きを読むには Pro プランへ」 → 「Pro プランを見る →」 クリック
3. **UpgradeModal が開く**（v31 で alert 化されていた問題が解決）
4. 「🎁 7日間 完全無料」「今すぐ無料で試す →」クリック → Stripe Checkout 画面へ
5. テストカード `4242 4242 4242 4242` で決済 → 自動でアプリに戻る
6. Webhook 受信で subscriptions.status = 'trialing' → useSubscription が反映 → **isSubscribed=true でフルビュー解放**

#### C. ログイン済 Pro ユーザー
- 何も変わらず FullView 表示

## 6. v32 で確立した UX/CVR 設計原則（永続ルール候補）

CLAUDE.md への昇格を検討すべき項目:

### 6-1. ロックコンテンツの設計原則
- **完全に隠さない** — 結論（センチメントバッジ）と冒頭プレビュー（80文字程度）は公開
- **続きはぼかしで存在を示す** — `filter: blur(4px)` + `mask-image: linear-gradient` で自然なフェード
- **CTA は摩擦を最小化** — 「30秒」「無料」「カード不要」等の安心ワードを明示

### 6-2. ログイン CTA の設計原則
- **Dead click（無反応ボタン）は厳禁** — 必ずログインモーダル or OAuth 画面に直結する
- **Google ボタンには Google アイコンを必ず併記** — 視認性 + 信頼性
- **シアン `#22d3ee` 塗り + glow `box-shadow: 0 0 12px rgba(34,211,238,0.30)`** で強調

### 6-3. 課金モーダルの設計原則
- **「無料」を主役に** — 価格より大きく、上に配置
- **トライアル期間 + 解約自由を併記** — 心理的障壁を最小化
- **メインボタンは1つ** — 月払いをデフォルト、年払いは控えめなセカンダリ
- **「その後 ¥X/月」は小さく副次情報として** — 即時の負担感を軽減
