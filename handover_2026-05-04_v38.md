## 1. Fix 1: CTA 文言の矛盾解消（最重要）

### 問題（v37 まで）
```
[ 🟢 無料で試す（登録30秒）]
✓ 登録不要で試せる　✓ 30秒で完了
```
**「登録不要で試せる」と「登録30秒」が同じ画面内で矛盾**。投資家ターゲットは読解力が高く、この矛盾を 1 秒で発見 → 全 LP の信頼を毀損する「Trust Cliff」現象。

### 修正後（v38）
```
[ 🟢 無料で始める ]
クレカ不要・Googleで30秒
```
- CTA 文言: 「無料で試す」→ **「無料で始める」**（常用性を暗示）
- 補助テキスト: 2 バッジ → **1 行**（クレカ不要 = B2C SaaS の最強キーワード / Google で 30秒 = メアド不要を暗示）
- バッジ装飾を削除して文章として読ませる（Apple 流）

### マーケター仮説
推測 CVR インパクト: **+30-40%**（信頼回復のみで）

## 3. Fix 3: 「今週の注目決算」セクション新設（最重要）

### 配置
「今日の注目」セクションの直後、機能紹介の前。

### 構造
```
─────[ ⚠️ 今週の注目決算 ]──────
        ↓ タップで事前分析

┌──────┬──────┬──────┐
│ NVDA │ WMT  │ COST │
│ あと │ あと │ あと │
│ 16日 │ 17日 │ 24日 │ ← amber #f59e0b
│2026- │2026- │2026- │
│05-20 │05-21 │05-28 │ ← 日付
└──────┴──────┴──────┘
```

### 技術構成
- **新規コンポーネント**: `UpcomingEarningsSection` ([LandingPage.jsx](frontend/src/components/LandingPage.jsx))
- **データソース**: `/api/calendar`
- **フィルタ**: `0 <= daysUntil <= 30`（仕様書では 7 日だが、実データを見ると 7 日以内が 0 件のため 30 日に拡大）
- **ソート**: `daysUntil` 昇順
- **取得**: 上位 3 件
- **表示**: ticker (20px) + 「あとX日 / 明日決算 / 本日決算」(amber) + 日付
- **クリック**: 親の `onTickerClick(ticker)` 呼び出し
- **データ 0 件 / エラー時**: セクション全体を非表示

### マーケター仮説
- **「今日の注目」 + 「今週の注目決算」** で「過去 + 未来」の両軸をカバー
- 「来週の TSLA / NVDA 決算前に判定したい」FOMO を喚起
- リピート率: 推測 **3x**（決算シーズンに毎日来たい）

## 4. Fix 4: Features/Pricing/FAQ 見出し削除

### 削除した 3 セットの見出し

| Section | 旧 |
|---|---|
| Features | `[FEATURES]` ラベル + `投資判断を、データで武装する。` |
| Pricing | `[PRICING]` ラベル + `シンプルな料金体系` |
| FAQ | `[FAQ]` ラベル + `よくあるご質問` |

### 削除理由
- **Hero と訴求重複**: 「投資判断を、データで武装する。」は Hero の「決算を見ても、買うべきか分からない」と訴求が二重
- **当たり前すぎ**: 「シンプルな料金体系」「よくあるご質問」は情報価値ゼロ
- **Apple 流**: コンテンツ（カード・FAQ アコーディオン）が主役。見出しは最小化

## 10. v38 で確立した永続ルール候補

CLAUDE.md への昇格を検討:

### 10-1. CTA 文言の矛盾は致命的
- 「登録不要」と「登録30秒」のような相反する語は同じ Hero 内に置かない
- ターゲットの読解力を信じて、矛盾は 1 秒で見抜かれる前提で文言を組む
- 信頼を損なうコピーは **CVR を 30-40% 落とす可能性**

### 10-2. 投資業界の色ルール
- 上昇 = **緑 (`#34ef81` 推奨)**
- 下落 = **赤 (`#f87171` 推奨)**
- 中立 = **シアン or グレー**
- 急騰 % をシアンで表示するのはユーザーの直感に反する

### 10-3. クリック可能シグナルの原則
- カードがクリック可能な場合、必ず **視覚的シグナル**を併記
- スマホでは hover が効かないため、`→` 矢印 / 「タップで...」マイクロコピー必須

### 10-4. 「未来の予定」セクションの重要性
- B2C SaaS で「**毎日見たい**」を実現するには、**「過去の動き」と「未来の予定」の両軸**が必要
- ニュースサイト系は全てこの構造（過去 = ヘッドライン / 未来 = カレンダー）

### 10-5. Apple 流: セクション見出しは最小化
- 「FEATURES / PRICING / FAQ」のような「セクションを説明する見出し」は基本的に不要
- コンテンツ（カード・アコーディオン等）が主役
- Hero でメインコピーを完結させ、以降のセクションは黙って見せる
