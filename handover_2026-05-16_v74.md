## 3. 触ると危険な箇所 (v74 新規 + 継承)

### v74 新規確定原則

**portfolio simulation の warmup 期間は構造的制約 (隠さない)**
- `earnings_evaluation` テーブルは 5 年分しかデータがない (FMP 5 年制限)
- `start_date - 365 days` まで pass_events_wide を fetch しても、 DB 自体が空のため初期 11 ヶ月は cash 100%
- LP narrative では 5y 全期間で「+87.42%」 と表記、 caption に「初期 12 ヶ月は eligibility lookback で cash 滞留」 を明記して透明性確保
- 隠すとデータ駆動 dogfood 時に逆効果

**`cum_return_pct` の意味は per-trade vs portfolio で異なる (混同禁止)**
- 旧 `kpis.cum_return_pct` = 「全 trade に同額分散投資した複利」 (理論値)
- 新 `portfolio.kpis.cum_return_pct` = 「月次リバランス portfolio の真の累積リターン」 (実値)
- 表示時は **どちらの数字か必ず文言で明示** (「1 銘柄あたり平均」 vs 「5 年累積・月次リバランス」)
- `feedback_chart_metric_consistency.md` 教訓 (Web / PDF / OG の chart 順序 + metric 完全一致)

**OG image のロゴと矢印は PDF と同じ規律**
- ロゴ背景円 = BG 同色 (slate-800 ではなく `_BG = (11, 17, 32)`)、 cyan 縁取り 78% opacity
- 矢印 = `_FONT_REGULAR` 36pt + slate-500 (100, 116, 139)、 Bold + `_TEXT_MUTED` 禁止
- セパレータ `・` は 3 セグメント均等の wide `　・　` で統一 (compound tight + 区切り wide のちぐはぐ禁止)
- 「明るい背景円で見やすく」 の旧 dogfood 方向は **撤回済**、 luxury 整合優先

**Cup-with-Handle 着手前の必読**
- `project_cup_handle_design.md` (本 handover で memory 化、 6 体合議 verdict + Phase 分割 + Premium 価値設計)
