## 🟡 重要知見 (永続化済、 v112 session 教訓)

### v112-5 / v112-9 大失敗 pattern ([[polish-iteration-roi-decay]] memory 起票済)
- **時間ベース motion (stagger / shimmer / cascade) 禁止** — scroll 中 frame で「未完成」 detect、 motion -4 regression
- **部分実装 (Hero だけ拡大、 下は既存) 禁止** — 「不均衡」 で全軸 regression、 procession idiom は full-redesign 前提
- **「装飾追加」 でなく「既存値強化」** 優先 (gold 1px → 2px、 padding 48 → 64px 等の修正系)
- **重要 4-5 section 限定** で signal、 全 section 拡張は逆効果

### vision-eval PDCA noise floor 判断
- Pane 3 overall 72.4 / Pane 4 70.0 / Pane 5 ~57 (Reading Mode、 locator-clipped、 採点不安定)
- 小幅 polish = ±1pt noise、 大胆 polish = -1〜-4 regression
- 構造再設計 (page 分割 / 別 route 切出し) へ shift 判断 SSOT

### Pane 5 自動採点不安定
- locator-based clip で領域 focus は動作 OK
- ただし NewsItem click 後 SSE 翻訳 race で **6 run 中 5 run FATAL** (Reading Mode mount されない)
- 真因仮説: backend SSE 翻訳の連続 fetch race / cache miss
- 解決: v113 で静的 page 化 → race 解消

### v112-2 / v112-3 教訓
- `usePortfolioPrices(allPortfolioTickers)` は holdings + transactions のみ fetch、 watchlist + selectedTicker は別途
- `setPrices(map)` は既存 state 破壊 → `setPrices(prev => {...prev, ...map})` merge 化必須
- priorityTicker 単独先行 fetch で 5 秒 → 1-2 秒短縮
