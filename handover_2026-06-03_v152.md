### Step 1: AI図解 視覚検証ハーネス (user 要望「目視検証できるように」)
**目的**: AI図解は Premium gate + 要ログインで本番 headless 描画不可。 → DiagramCard をモック data で単体レンダー
する headless プレビューを作り、 改修のたびに screenshot を撮って `SendUserFile` で user に送る (デプロイ + dogfood 不要)。
**技術 (検証済の approach)**:
1. `frontend/preview.html` (entry) + `frontend/src/preview-main.jsx`: `<MotionProvider><DiagramCard data={FIXTURE} ticker="MSFT" /></MotionProvider>` を render + `import './index.css'`。 ⚠️ MotionProvider (`src/components/MotionProvider.jsx`) で wrap 必須 (m.* が initial opacity:0 固着、 [[feedback_pane_error_boundary]])。
2. `frontend/vite.preview.config.mjs`: base:'./' (file:// 用 相対パス) + rollupOptions.input=preview.html + outDir='.preview-dist'。 本番 build (base:'/') は触らない。 `npx vite build --config vite.preview.config.mjs`。
3. `frontend/scripts/snap-diagram.mjs` (visual harness 例外準拠: headless / 55s timeout / .visual出力 / HTTPサーバなし): `chromium.launch({ headless:true, args:['--allow-file-access-from-files'] })` ← file:// の ES module CORS 回避が鍵。 `page.goto('file://<abs>/.preview-dist/preview.html')` → `page.screenshot({ path:'.visual/diagram.png', fullPage:true })`。
4. FIXTURE (DiagramCard `data` prop = visualize response): 必須 field = headline / summary / overallPass / verdict / passCount / totalCount / conditions[] / valuation / trends[](metric+data[]) / operatingMargins / segmentSummary{date,segments[{name,value_b,yoy_pct}]} / guidanceExtracted{source_type:'transcript',narrative_only:true,narrative_jp,source_quote,source_label,...} / strengths[] / risks[] / bullCase[] / bearCase[] / investorQuestions[] / businessFlowSteps[] / fcfTrend[] / capexTrend[] / fcfYield / capitalReturn{dividend,buyback} / companyName / period / consensusSource / gaapAdjustment。 → 忠実な値は本番 `POST /api/visualize/MSFT?years=4` に realistic enrichedData (DetailReport.jsx:563 参照: ticker/company_name/verdict/conditions_detail(JSON)/periods[]/guidance/beat_miss) を送って capture が確実。 or 手書き fixture を `frontend/.visual/diagram-fixture.json` に。
5. 検証: snap → .visual/diagram.png を `SendUserFile` で user へ。
