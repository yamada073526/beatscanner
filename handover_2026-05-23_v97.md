## 4. 永続教訓 (v97 で確立、 memory anchor 候補)

### A. Vision-eval scorer の frame count 依存 bias (新規 memory: feedback_vision_eval_frame_count.md)

scorer で ticker 切替時に accordion default 状態で frame 数変動 (5 vs 8) → motion 軸基準ばらつき。 anchor 画像 base scoring と frame 統一が次セッション必須。

### B. FMP API path Legacy 認定の silent fail (新規 memory: feedback_fmp_api_legacy.md)

`/api/v3/*` `/api/v4/*` は 2025-08-31 以降 Legacy 認定で 403。 `safe_fmp_get` の Limit Reach フォールバックで silent fail、 数ヶ月気づかず。 四半期ごと probe script で health check 推奨。 v97 で `/stable/*` 移行済 (segment endpoint)、 他 v3/v4 endpoint も grep で全 audit 必要。

### C. CLS 防止は「上方 section の高さ envelope」 が最優先

user 「scroll 中ガクツキで Pane 3 開くのが面倒」 は v97 で判明した最大 retention 因子。 真因: 上方 lazy load section が data 到達で 100-300px 高さ増加 → 下方位置押し下げ → 現在見ている要素が画面外に消える。
対策: 大型 chart / image 系 section に **必ず minHeight envelope** を設定、 loading でも data でも同じ height で固定。 これは vision-eval (静止画) では検知できず、 user dogfood でしか発見できない pattern。

### D. AI scoring の session 内 drift bias

3 run mean でも variance ±4 残る。 「初回甘め、 2 回目以降 厳しめ」 pattern。 anchor 画像 base + fresh session pool で対策。

### E. Sprint 連続追加で sanitize 漏れ pattern

LLM tool schema 拡張時、 backend `_call_llm` return mapping / frontend sanitize / frontend表示 component の 3 箇所同時 update が必要。 v97 で Sprint H6 残債として ProfileCard の sanitize 漏れ発覚。
