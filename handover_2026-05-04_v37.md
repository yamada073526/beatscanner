## 1. Fix 1: Hero コピーを「痛み→解決」型に変更

### 変更前 (v36)
```
β版・招待制プレビュー公開中
決算を、瞬時に
読み解く。
売上・EPS・CF、AIが自動分析。
[ 無料で試す（登録30秒）]
```
- 機能説明型 → 「自分のため感」が湧かない
- β バッジが「招待制」と書きながら誰でも登録可能 = 虚偽訴求

### 変更後 (v37)
```
決算を見ても、
買うべきか分からない。
プロが使う5条件で、3秒で判定。
[ 無料で試す（登録30秒）]
```
- **痛みのキーワードを直撃** (投資家が心の中で呟いている言葉そのもの)
- **権威性 (プロが使う) + 具体性 (5条件、3秒)** で解決策を提示
- β バッジを削除して虚偽訴求リスクを排除

### マーケター仮説
「自分の悩みを言語化してくれている」感覚で Hero での留まり時間が伸び、CVR が **2x** になる可能性。

## 2. Fix 4: 「🔥 今日の注目」セクション新設

### 配置
Hero 直下、機能紹介の前。**未ログインユーザーが最初に目にする情報**。

### 構造
```
─────[ 🔥 今日の注目 ]──────

┌──────┬──────┬──────┐
│ MSTR │ TSLA │ NVDA │
│ +7.08%│ +5.21%│ +3.45%│
│ ビット…│ EV需要…│ AI半導…│
└──────┴──────┴──────┘
   ↑ クリックで即座に分析実行
```

### 技術構成
- **新規コンポーネント**: `TodayHotSection` ([LandingPage.jsx](frontend/src/components/LandingPage.jsx))
- **データソース**: `/api/movers` の `gainers[]` 上位 3 件
- **表示項目**: `ticker` / `+pct%` / `keyword` (20文字)
- **クリック動作**: 親から渡された `onTickerClick(ticker)` 呼び出し
- **ローディング中**: pulse スケルトン (`@keyframes pulse`)
- **エラー時**: セクション全体を非表示 (`movers === 'error'` で `return null`)

### App.jsx 側の接続
```jsx
<LandingPage
  onSignIn={signInWithGoogle}
  onProCheckout={() => startCheckout('monthly')}
  onTickerClick={(t) => {
    setTicker(t);
    runAnalyze(t);
    setActiveTab('judgment');
  }}
/>
```

### マーケター仮説
- 「契約させて終わり」→「**使えるダッシュボード**」への格上げ
- リピート訪問率 **3x** (推測)
- 未ログインで Aha モーメント体験 → 後続の登録 CVR が **1.5-2x** (推測)

## 4. Fix 2/3/6/7: ノイズ削減と信頼/価格訴求の最適化

### Fix 2: プルーフチップ 3 個削除
Hero 下の「✓ 5条件即時判定 / ✓ AI市場分析 / ✓ 株価チャート連動」を完全削除。
機能セクションで詳述するため重複。

### Fix 3: CTA 直下バッジを 3 → 2 個
| 旧 | 新 |
|---|---|
| ✓ クレカ不要　✓ 30秒で完了　✓ 3銘柄まで永久無料 | **✓ 登録不要で試せる　✓ 30秒で完了** |

「3銘柄まで」の制限訴求を削除して**萎縮効果を回避**。「登録不要」を主役に。

### Fix 6: データソース表記復活
v36 で削除した TrustSection を 1 行で復元 (Pricing と FAQ の間):
```
Powered by Financial Modeling Prep · Yahoo Finance · Anthropic Claude
```
信頼シグナルとして必須だが、邪魔しない超控えめな配置。

### Fix 7: コーヒー1杯アンカリング
Pro カードの ¥980/月 直下に追加:
```
1日あたり約¥33 — コーヒー1杯より安く
```
B2C SaaS の価格表現ベストプラクティス。価格を「1日換算 + 比較対象」で表現することで**体感価格が大幅に下がる**。

## 8. v37 で確立した永続ルール候補

CLAUDE.md への昇格を検討:

### 8-1. LP の Hero は「痛み → 解決」構造
- 1 行目: ターゲットの心の声 (痛み)
- 2 行目: 解決策 (権威性 + 具体性)
- 機能説明・サービス名訴求は二の次

### 8-2. CTA は「登録」より「試す」
- 「登録30秒」「無料で始める」より「**試してみる**」「**結果を見る**」の方が心理障壁が低い
- Aha モーメント先行戦略 (Notion / Linear / Figma 全社採用)

### 8-3. 機能カードは「説明文」より「実 UI モックアップ」
- 言葉で説明するより、CSS で実 UI を再現する方が説得力が圧倒的に高い
- `FeatureCard` の `mockup` prop パターンを横展開可能

### 8-4. 「今日の」要素を LP に必ず置く
- B2C SaaS で再訪率を上げる最重要要素
- 静的 LP は「契約させて終わり」、動的 LP は「ダッシュボード化」
- API がある全機能は LP のサンプル表示も検討

### 8-5. 価格は「1日換算 + 比較対象」で表現
- ¥980/月 → 「1日¥33・コーヒー1杯より安く」
- 体感価格を大幅に下げる効果
