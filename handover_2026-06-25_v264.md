## ✅ 本セッションの決定(SPEC v2 の核心)

mockup を正本とした結果、当初の素朴な「beat/cfps を共に earnings_pass の任意トグル」案は **3 点ズレ**ており、3 体合議(ui-designer/frontend-architect/qa-dogfooder、全員「折衷」一致)+ user 判断で以下に確定:

| 条件 | preset | 形式 | default |
|---|---|---|---|
| `latest_beat` | **新高値ブレイク** | **gate 必須(常時ON)** | — |
| `eps_3y_rising` | 決算合格・継続性 | binary toggle | OFF |
| `rev_3y_rising` | 決算合格・継続性 | binary toggle | OFF |
| `cfps_3y_rising` | 決算合格・継続性 | binary toggle | OFF |
| 3 段階精度 UI(mseg) | — | **出さない** | — |

**確定根拠**:
- beat は earnings_pass でなく **新高値ブレイク**(mockup L250)。テクニカル×決算の交差=原則4。
- eps3/rev3 も **3 点セット**で含める(mockup の決算合格は trio・配線ほぼ無料・user 確定で当初 Q3「cfps のみ」を覆す)。
- **default OFF**(実データ cfps true率≈28% で default-ON だと決算合格が 72%減=離脱。mockup の ON は mock data 前提=改善点)。
- **段階 UI は出さない**(boolean データで 3 段階=偽の段階=Trust Cliff・全員一致)。graded 化は backend backlog。
