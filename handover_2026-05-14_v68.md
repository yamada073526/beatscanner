## 4. 触ると危険な箇所 (CLAUDE.md + v67/v68 追加)

### v68 追加
- **window event `bs:transactions:changed`** (`useTransactions`): 同 user 内で複数 instance が ある場合の state 同期に必須。addTransaction / updateTransaction / removeTransaction で必ず dispatch、listener 側で reload を呼ぶ。新しい transactions 系 hook を作る場合は同 event を購読すること。
- **`--surface-card` は class、CSS custom property ではない**: modal / card 系 component で `var(--surface-card)` と書くと **fallback なし → 透過にフォールバック**。**`var(--bg-card)` を使う**。本セッションで TransactionEntryModal が貫通バグを踏んだ教訓。
- **CLAUDE.md UI 文言ポリシー**: 「じっちゃま」は内部 (comment / file name / docs) のみ、UI には「ファンダメンタル5条件」「独自プロトコル」を使う。`.claude/hooks/pre_edit_trust_cliff.sh` で機械的検査。
- **PortfolioPaneSection の effectiveHoldings 切替**: transactions が 1 件以上あれば transactions ベース、空なら holding_lots ベース。これにより後方互換維持 (まだ取引登録していない user は壊れない)。
- **transactions.fx_rate 凍結書き込み**: trade 時点で fx_rate を transactions 行に固定値書き込み (Stripe/Wise 方式)。後から forex_rates の rate を変えても historical P/L が動かない (会計 audit 性)。Phase 2.5 で USD/JPY 有効化予定。
