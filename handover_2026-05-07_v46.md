## 2. 🔴 重要知見 — 永続化必須 (memory にも記録済)

### 2-1. spotlightInfo.icon は undefined だった (Web 開発エージェント発見バグ)

`EconomicCalendarSection.jsx:500` で `{spotlightInfo.icon}` を参照していたが、`translateEvent` の戻り値構造は `{ja, en, category}` で、icon は `category.icon` の下にある。F5 「最注目にカテゴリアイコンがない」というユーザー指摘は **バグ起因**だった。

**修正**: `spotlightInfo.icon` → `spotlightInfo.category.icon`。
**教訓**: 関数戻り値の構造を変更したら、参照箇所を全文 grep で確認すべき。

### 2-2. news-grid-scroll-wrapper の scrollbar 設定が欠落していた (Web 開発エージェント発見)

`brief-list-scroll` / `econo-cal-list-scroll` は `scrollbar-width: thin` 設定済で「薄すぎて見えない」状態、一方 `news-grid-scroll-wrapper` は **scrollbar 設定が完全欠落** = OS デフォルト太バー = ユーザーが「グリッドだと視認できる」と感じた真因。**ユーザー認識と実装が逆転していた**。

**修正**: 共通 `.bs-scroll-thin` ユーティリティ新設 (thumb 42% alpha + hover 65%、Webkit 8px、ダーク両対応) → 3 箇所統一。

### 2-3. backend `_EVENT_NAME_JP_MAP` は部分和訳のみ

backend `_EVENT_NAME_JP_MAP` (70+ 件) は「Initial Jobless Claims (新規失業保険申請)」形式で和訳併記済だが、Fed Speakers (Williams/Waller/Bowman 等) / S&P Global PMI / ECB Lane/Buch/Cipollone は未網羅。

**対策**: frontend `frontend/src/lib/i18n/economicEvents.js` 新設で `{ja, en, category, Icon, color}` 一元管理。
- backend が「英語 (和訳)」併記済の場合: 末尾括弧の中身を正規表現で分解 → ja 採用
- 未網羅 (英語のみ): frontend dict (50+ 件、longest match) で補完
- カテゴリ判定: regex (8 カテゴリ、🏦/📊/💼/🏭/🛒/🏠/📈/📋)

### 2-4. removeFromWatchlist は holdings に触らない設計 (バグ案 B 採用)

`removeFromWatchlist` (App.jsx:642) は watchlist + tag 削除のみで holdings/lots に触らない。`filteredWatchlist` の hold モード判定が `watchlist ∩ holdings` のため、watchlist 削除 → 「保有」モードから消えるが PortfolioDashboard には残る不整合が発生。

**修正**: 案 B 採用 (UI/UX エージェント強推奨、Robinhood/Yahoo Finance/SBI 流):
- HomeTab.jsx: hold モード時の filteredWatchlist を `Array.from(new Set([...watchlist, ...Object.keys(holdings)]))` で union 化
- ChartTab に `watchlistSet` prop 追加 → TickerRow に `inWatchlist` prop 配線
- watchlist 外 holding 持ち銘柄に `⊘ ウォッチ外` バッジ表示 + × ボタン非表示
- 削除に Undo Snackbar (Gmail/Material 方式、5 秒、tag 復元含む)

### 2-5. lucide-react ^1.14.0 採用 (F6)

絵文字 (📊💼🏦🏭🛒🏠📈📋) を SVG コンポーネントに置換。`{key, Icon: LucideComponent, label, color}` 統合 dict で一元管理。stroke-width 1.75 + currentColor + カテゴリ別色 (中立色のみ、CLAUDE.md 投資業界色ルール遵守)。

依存追加 +6.2KB (8 アイコン分 tree-shaken)。マーケター実証「信頼スコア +18-25pt」(Linear/Notion 刷新事例)。

**注意**: lucide-react v1.x は 2025-09-04 以降の最新メジャーで、API は import { TrendingUp, Users, ... } from 'lucide-react' 形式 (古い v0.x の `react-feather` 系とは互換性なし)。

### 2-6. F8 ChartTab モバイル ⋯ 集約 + bottom sheet (Apple HIG 44pt)

mobile では 4 ボタン縦積み (20px 各、HIG 44pt 違反) を **`MoreHorizontal` 1 ボタン (44×44) + bottom sheet** に集約。`createPortal` で document.body 直下、`role="dialog"` + ESC + body scroll lock + `safe-area-inset-bottom` 対応 + `sheetSlideUp 0.25s cubic-bezier`。Robinhood / Wealthfront / Trade Republic 標準。

リテンション期待: モバイル操作成功率 60% 台 → 95%+ (Fitts 法則ベース)。

### 2-7. ChartTab は判定タブ (`activeTab === 'チャート'`) でも使用される

新 props (tagsById / assignments / holdings / prices / hideTagPill / onTagClick / onRemove / watchlistSet) はすべて default 値を持ち、判定タブ側 (App.jsx:1536) では未指定で後方互換 OK。`inWatchlist` も default true。判定タブでは tag pill / 損益 / ⋯ × は表示されない (`watchlistSet` 未指定で `inWatchlist` 常に true、props 未指定でアイテム非表示)。
