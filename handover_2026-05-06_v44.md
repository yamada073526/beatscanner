## 5. 🔴 重要知見 — 永続化必須 (memory にも記録済)

### 5-1. `railway up` は worktree から実行不可
**memory: `feedback_railway_worktree.md`**

`railway up` を git worktree (`.claude/worktrees/<name>/`) から実行すると、worktree の作業ディレクトリではなく **main ブランチの git tree が upload される**。snapshot SHA が 4 回連続で `cc8a4f3c...` で一致することで判明。

→ **必ず main repo (`/Users/yamadadaiki/Projects/beatscanner/`) から `railway up`**。worktree branch の変更は `git merge --ff-only` で main にマージしてからデプロイ。

### 5-2. 開発ツール使用時はその都度通知
**memory: `feedback_devtool_narration.md`**

§4-A で導入した開発ツール群 (slash commands / hooks / skills / GH Actions) を**起動・利用するたびに**、ユーザーへ短く通知 (1-2 行)。Trust 構築フェーズのため。

### 5-3. iOS Safari の date input カレンダーアイコン
- `::-webkit-calendar-picker-indicator` への `filter: invert(1)` / `color-scheme: dark` は iOS Safari で部分的にしか機能しない
- **SVG background-image 方式が確実**: ネイティブ indicator を `opacity: 0 + position: absolute` で透明化 → input 自体に `background-image: url("data:image/svg+xml...")` で stroke カラーを theme に合わせる
- Lucide / Heroicons / shadcn-ui 標準パターン

### 5-4. Supabase migration の 2 ファイル順次パターン (v42 §5-2 継承)
本セッションで Holdings X-2-B + Notifications Y-3-A の合計 4 ファイル適用済:
```
1. 2026-05-06_holding_lots_x2b.sql + grants
2. 2026-05-06_notifications_y3a.sql + grants
```
Supabase SQL Editor で create したテーブルは `GRANT select, insert, update, delete ... TO authenticated` を別ファイルで明示しないと RLS policies があっても authenticated ロールが書き込めない (silent permission denied)。

### 5-5. FMP 無料プランの fallback chain
fmp-api-retry skill (`.claude/skills/fmp-api-retry/SKILL.md`) に永続化:
```
FMP (primary) → AlphaVantage → yfinance → graceful degradation (空配列 / null)
```
本セッションで `/api/portfolio-history` に yfinance fallback を実装。同じパターンを他 endpoint にも適用予定 (例: `/api/guidance/{ticker}/quarterly-history` は現在 FMP のみで 404 多発、要 fallback 追加)。
