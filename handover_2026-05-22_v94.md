### Phase 2.9 Sprint 1 真因 fix (commit `b70ae41`)

user dogfood 4 連続「全部変わってない」 を受けて snap-debug-pane29.mjs で 本番 DOM + computed style 取得 → **4 件の真因確定** + 修正:

#### #1 + #5-a アナリスト視点 / 会社概要 左右クリッピング (Phase 2.6/2.7/2.8 で 3 回試行も解消せず)

- **真因確定**: `AccordionSection.module.css` の `@keyframes acc-expand` に **`forwards` で残る `clip-path: inset(0 0 0% 0)`** が tier-m-glow ::after の outer halo を bounding box で clip
- Phase 2.6 (root `.section` overflow:visible) / Phase 2.7 (m.div state-aware overflow) / Phase 2.8 (`.panel` overflow:hidden 削除) は **すべて overflow 系の修正**、 clip-path forwards 持続は別 clip context として残っていた
- **修正**: keyframe 終了 frame を `clip-path: none` に置換 (99% trick で interpolation 回避)
- **メタ教訓**: 「clip-path forwards 持続」 = `feedback_clipping_root_cause_chain.md` の wrapper chain SOP に追加候補 (handover v94 §3 起票候補)

#### #2 GuidanceCard 「スクロール発光弱い」 (Phase 2.6/2.7/2.8 で 3 回試行も解消せず)

- **真因確定**: `.tier-m-glow[data-halo-fired="1"]::after` は `display:none` で fire 後に halo が完全消失、 wrapper baseline も border 1px のみで「ambient halo presence」 がゼロ
- 5 section すべて (FiveConditions / Guidance / EarningsHistory / Analyst / QuarterlyHistory) で同じ問題、 GuidanceCard は内部 cyan accent 密度が他 section より低いため最も「弱く」 見えた
- **修正**: `.tier-m-glow[data-halo-fired="1"]` に **inset 14% + outer 10% accent box-shadow** を baseline として持続 (Aman 廊下の常夜灯比喩、 sweep 後の余韻)。 dark mode 強化版 (22% / 16%) あり
- 副次効果: 5 section 全部の fire-後 ambient halo が均質化

#### #3 accordion 展開時「キラッとしない」 (Phase 2.8 #3 で setTimeout なしで実装)

- **真因確定**: `AccordionSection.jsx:192` で `{isOpen && (children)}` → accordion closed 中は **AnalystPanel / QuarterlyHistoryTable が DOM に未 mount**。 onOpenChange 同期 fire 時点で `haloTriggerRef.current` は null、 fire しても no-op
- **修正**: `JudgmentDetail.jsx` で `setTimeout(() => analystHaloTriggerRef.current?.(), 500)` で trigger 遅延 (spring animation 約 400ms + mount + useEffect 完了を待つ)

#### #4 NewsPanel アクセントバー (Phase 2.8 で list view 漏れ + scaleY なし)

- **真因**: 旧 `.news-list-card::before` は static width 拡幅 (2.5px → 4px) のみで IRLinks の `scaleY(0→1)` slide-in animation を持たない。 user 「枠と一緒に動くが伸びるアニメーションしない」 指摘
- **修正**: `scaleY(0)` initial + hover で `scaleY(1)`、 `.is-open` でも visible (selected highlight 維持)。 IRLinks SSOT を完全 mirror

## 3. v94 で確立した永続教訓 (memory 起票候補)

### A. clip-path forwards 持続による outer halo clip (Phase 2.9 #1 + #5-a 真因)

memory 名候補: `feedback_clip_path_forwards_trap.md` (or `feedback_clipping_root_cause_chain.md` に追記)

- `@keyframes ... forwards` の終了 frame に `clip-path: inset(X)` が残ると、 absolute positioned ::after pseudo の outer halo を bounding box で clip
- 99% trick で 1 frame 前に `clip-path: none` に escape する pattern
- 関連 anchor: [[feedback-clipping-root-cause-chain]] (Phase 2.8 で起票)、 [[glow-elevation-postmortem]] (v54-v59 発光バグ)

### B. AccordionSection mount timing race (Phase 2.9 #3 真因)

memory 名候補: `feedback_accordion_mount_race.md` (or [[feedback-halo-sweep-accordion]] に追記)

- `{isOpen && (children)}` pattern で children mount は **next render cycle**
- `onOpenChange(isOpen=true)` 同期 fire 時点で child ref は null
- setTimeout (animation 完了時間 + buffer) で deferred trigger が必須
