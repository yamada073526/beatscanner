## 3. 触ると危険な箇所 (v105 + v106 教訓)

### v105 から継承
- sticky 検索バー / 発光系 / VITE_ARG/ENV / Hallucination Guard / FMP /stable
- ReadingMode 全面 Overlay / 章 2 ChapterTabs 二重表示防止 `!isV3 &&`
- App.jsx lucide-react import (v105 で追加完了)

### v106 新規教訓
- **章 3 ChapterTabs と既存 InsightsPanel section の二重表示防止**: isV3 ON 時に InsightsPanel section を `{!isV3 && ...}` で skip。 Phase G Phase 4 で章 2 QuarterlyHistoryTable に対して行ったのと同 pattern
- **halo sweep ref は accordion onOpenChange ベース、 tab interface とは meta が違う**: AnalystPanel に haloTriggerRef を渡すと tab 内で「accordion open」 event を待つことになり発火しない。 v106 v1 では渡さない簡素化、 dogfood 後に「tab 切替で halo 演出が欲しい」 となれば ChapterTabs に `onActiveMount` callback 追加で対応
- **ch3tab URL state は ch2tab と独立 setter**: 章 2 と章 3 で別 URL param、 同じ pattern で簡素化。 将来章 4 (もし reorg するなら) も ch4tab で独立 management
- **section ID `-v3` suffix**: ChapterTabs 内の AnalystPanel / InsightsPanel section ID は `-v3` suffix を付けて旧 accordion `sec-analyst` / `sec-insights` と区別。 anchor link や cmd-palette の `expandedSections` set 機能と干渉しないため

## 5. 永続 memory anchor 候補 (v106 新規)

- **feedback_chapter_tab_dual_pattern.md** (新規): 章 2 (3 tab) + 章 3 (2 tab) で同 ChapterTabs primitive を 2 章で展開した pattern。 ch{N}tab URL state を独立 setter で管理、 isV3 共通 flag で「v3 mode 全体切替」 を 1 つの flag で達成。 将来章 4-N も同 pattern で拡張可
- **feedback_halo_sweep_vs_tab.md** (新規): halo sweep は accordion onOpenChange event ベースで設計、 tab interface とは event meta が違うため tab 内で渡しても発火しない。 解決案: ChapterTabs に `onActiveMount(tabKey)` callback 追加 + tab 切替で外部 ref を trigger
- **feedback_section_id_v3_suffix.md** (新規): feature flag mode で同 component が複数 section ID で render される場合、 `-v3` 等の suffix で区別する pattern。 cmd-palette / expandedSections set / anchor link の干渉防止

### 既存 anchor 更新候補
- `feedback_chapter_tab_pattern.md` (v105 で追加): 章 3 ChapterTabs 追加で「単独 section だけでなく、 章レベルで複数 component を集約する idiom」 と説明拡張
