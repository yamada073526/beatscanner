## 0. v2 改訂の経緯（最重要・なぜ叩き台5案を捨てたか）

他 AI 提示の叩き台5案（パーフェクト・ゲーム / 新波動入り / IPOサバイバー / Rule of 40 SaaS / 全天候型配当）を、**原典 SSOT（じっちゃまプロトコル投資哲学10箇条 + 絶対6条件）に逐語照合**した結果、構造的欠陥が判明:

### 10箇条 × 叩き台5案 カバレッジ
| 箇条 | 5案のカバー |
|---|---|
| ①良い決算 | **過剰**（4案集中） |
| ②分散 | ほぼ抜け（P5の「配当=分散」は正当化不成立） |
| **③業界No.1を先ず検討** | 🔴 **完全抜け** |
| ④過去最高値が好機 | P2のみ |
| ⑤金利に注意 | 🔴 抜け（ただし screener 外＝地合いbanner が適切） |
| ⑥CFマージン15%+3期連続増 | P1のみ |
| **⑦テーマ賞味期限→新天地** | 🔴 **完全抜け（最重要欠落）** |
| ⑧⑨⑩アノマリー/規律/売り | 抜け（screener 外＝売りルール側） |

### 判定（独立2体 Opus レビューが収束）
- **却下 P4 Rule of 40**: 10箇条に根拠ゼロ。サブスク比率/リテンションは FMP 標準 field 無し＝LLM抽出 hallucination risk。原典・実装の両面で最弱。
- **却下 P3 IPOサバイバー**: IPO選好は10箇条に無い（オニール N由来）。第7条「新天地」はセクター/テーマであって上場時期ではない。毎朝の人力作業を代替しない。
- **却下/再生 P5 配当NISA**: 「配当=第2条分散」は正当化不成立（第2条はポートフォリオ構築の戒め）。価値は「業界リーダー」部分のみ→第3条プリセットへ再生。
- **維持 P1/P2**: 絶対6条件（P1）・第4条新高値（P2）は原典忠実。ただし P2 はオニール装飾（Cup-Handle/出来高+40%）を削り「新高値主・テクニカル従」に純化。

### Reviewer B の構造的洞察（user 違和感の正体）
叩き台5案は**全部が「静的 universe screening（数日に一度の R6 作業）」**。だがじっちゃま実践者の**毎朝の重心は ①決算合否トリガー（R1）・②保有/WL の崩れ監視（R5・売り側）**。5案中4案が「毎日中身が変わらない」＝**原則2「毎日開きたくなる」に弱い**。→ 改訂版は重心を **「静的フィルタ」から「昨晩〜今日の差分を出す動的トリガー」** へ移す。

### 3. 今の主戦場（セクター物色）★新規 backend ロジック
```
新ビュー: セクター別RS（rs_vs_spy_pct を sector で median/mean 集計）をランキング
        → RS が直近改善方向に転換したセクター（=新天地）を上位表示
```

1. **「決算合格」は「昨夜」でなく「直近の決算シーズン」框定**: 単日決算だと対象が過少。**最新四半期＝過去90日窓**（既存 `earnings_evaluation` の 95日窓と整合）で絞る。UI に「対象: 直近の決算シーズン（過去90日 / 例 2026 Q1）」の period chip。決算シーズン中は日々追加されていく差分性は維持。

## 7.7 モックアップ v4 で確定した設計（user dogfood 第3巡 2026-06-22）

### ビジュアル
1. **ゴールド線画アイコン**（模範解答 PDF「AI-Driven School 第8回」様式に準拠）: 線画 SVG（stroke 1.6px）を**角丸タイル + 淡色背景（フラット・非発光）**に収める。色は cyan → **gold**（`--color-gold #d4af37` / `-mid #f4cd5d`、ALLOWED-HEX 登録済 handover v69）。active preset の arrival も gold glow（`0 0 0 1px rgba(212,175,55,.32), …`）に統一。
2. **アニメーション**: 精度 segmented に**スライドする thumb**（transform .28s ease-out-expo）、各条件トグルの knob slide、件数変化時に**pop アニメ**（scale 1→1.28→1）、refine パネルの max-height 開閉トランジション。

### アドバンスド = 個別緩急（ファジー条件設定）
3. 精度プリセット（緩い/標準/厳しい）の上に、**各条件を個別に 緩・標・厳・最厳 で設定**できる mini-segmented を追加（例「EPS は最厳 +100%、CFPS は緩め」）。初心者=プリセット1クリック、中級者+=個別緩急。**「完成形からの差分編集」**が汎用スクリーナー（ゼロから組む）に対する効率優位。

### アドバンスドの課金ゲーティング（funnel-cro Opus 合議反映・Trust Cliff 厳守）
4. **線引き**: **精度プリセット3段は Free で 100% 動作**（LP「探す手間ゼロ」と整合・有料化しない）。**個別緩急（アドバンスド）のみ Pro**。
5. **離脱でなく課金動機にする3手法**: ①**件数ちら見せ**（「標準23銘柄。EPS を +50% に締めると 6 銘柄」を Free にも計算表示、銘柄リストはロック）②**設定UIちら見せ**（mini-segmented を disabled で見せる）③**保存ゲート**（カスタム設定の保存＝Pro、将来 Signature nightly push のスキャン条件に直結 → 原則4「人力の全代替」導線）。
6. **mockup で Free/Pro 切替**を用意し、Free 時にロックバー＋件数ちら見せ、Pro 時に個別緩急フル操作を確認可能。

### 競合差別化（課金動機）
7. **核**: Finviz / TradingView / 楽天スーパースクリーナー等は「フィルタを**組む**」だけで**正解の閾値（処方箋）を持たない**。BeatScanner は**じっちゃま＋オニール原典の閾値（EPS +20/25/50/100% 等）をプリセットに焼き込み、1クリック適用 → 個別微調整**。IBD MarketSurge は本家だが英語・$40前後。**「勝ち筋が最初から入っているスクリーナー（日本語・決算駆動・原典内蔵）」**が差別化。
8. **価格方向性**（実証は A/B）: Pro **¥1,480–1,980/月**（MarketSurge/Finviz Elite $40前後より明確に安く、文脈適合は上）、Premium ¥2,980前後、Signature ¥10k（nightly push 維持）。
9. **§38/§5 訴求指針**: 主語を常に**「あなたの設定 / 手間の削減」**に置き、「儲かる/勝てる/最強/必ず」を出さない（「人力の代替」は事実ベースで訴求可）。LP/ペイウォール実装時は `funnel-cro`（Trust Cliff 7項目）+ `hallucination-guard` を必ず通す。

## 7.8 モックアップ v5（プロ UI/UX 3体レビュー反映 2026-06-22）

UI/UXデザイナー + モーション/フロントエンド + QA の独立3体レビューで v4 改修プランを検証し、以下を確定。

### ① アイコン / 選択表現（gold 一本化）
- **タイル = シアン下地（不変）**。**アイコングリフ = 非選択シアン / 選択中ゴールド**（一目で選択判別）。**hover = アイコンがゴールド発光**（`drop-shadow`）+ カード lift + 微 gold glow（v1-3 の hover 演出を復活）。
- 選択中カードの arrival も **gold へ一本化**（cyan glow と gold icon の併用＝過飾を回避、レビュー1指摘）。cyan はタイル下地のみに限定。focus-visible ring も別途必要（accessibility）。

### ② アニメーション（技術修正）
- **トグル knob 非アニメ = 毎回 `innerHTML` 全再描画が原因**。→ **in-place class toggle**（DOM 再利用）に修正、件数等は **`textContent` 限定**部分更新。knob の transform transition が発火。
- パネル開閉 = **`grid-template-rows:0fr↔1fr`**（max-height hack の正統後継・開閉とも滑らか・Chrome107+/FF116+/Safari16.4+）。**開 350ms / 閉 200ms 非対称**（高級UI標準: 閉は短く）。
- easing 分離: パネル=expo `(.16,1,.3,1)` / knob・thumb=Material `(.4,0,.2,1)` / 件数pop=spring `(.34,1.56,.64,1)`。全要素 expo は逆効果（レビュー2）。
- **`prefers-reduced-motion: reduce` で全 transition/animation 無効化**（WCAG 2.3.3・前庭障害配慮）必須。

### ③ アドバンスド（全条件緩急・二値は必須ゲート明示）
- **ラダーのある条件はすべて 緩/標/厳/最厳 で個別設定**（CFマージン ≥10/15/20/25、連続性 直近正/3期/3期+加速、EPS +20/25/50/100、ROE、RS、出来高…）。各 mini-segment に**閾値数値を併記**（「緩 +20%」、レビュー1必須指摘）。
- **真の二値（CFPS>EPS=粉飾防止 / 52週高値更新 / 機関保有増 等）は緩急化せず「必須ゲート」バッジで明示**（無理に4段化しない、3体合議一致）。条件を**グループ化**（成長性/収益の質/モメンタム/需給）で認知負荷減。
- 精度プリセットから個別変更したら**「カスタム」タグ**を表示（状態の見える化、レビュー1）。

### ④ 課金ゲート / コピー（Trust Cliff 回避）
- ロックバーは**アドバンスドを触った時のみ**表示（常駐しない）。**「全件見られる」「解放」「ロック」の語は撤廃**（list 可視を約束する Trust Cliff 語感、QA指摘）。
- 新コピー: 「プリセットで **N 銘柄**に絞り込み中。条件を個別に詰めるには Pro へ」（Free 価値を先に肯定）。**Pro 訴求の重複（fhint 側）を削除**。
- 精度プリセット3段は Free で100%動作（不変）。
