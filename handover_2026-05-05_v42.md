
## 5. 🔴 重要知見 — 永続化必須

### 5-1. Clearbit Logo API は廃止 (HubSpot 買収後)

- **影響**: 旧来「ロゴと言えば Clearbit」が完全に使えない
- **対応**: TradingView SVG (主要銘柄) + FMP image-stock (網羅性) + 頭文字円
- **既存コード**: 過去の `getTickerDomain` 等は使わない、`tickerSlugs.js` の `getTickerTvSlug` を使う

### 5-2. Supabase で SQL Editor 経由作成テーブルは GRANT 別途必要

- **症状**: ユーザーが操作すると `permission denied for table {table_name}` エラー
- **原因**: SQL Editor で `create table` した場合、anon/authenticated に自動 GRANT 付与されない
- **対応**: `grant select, insert, update, delete on public.{table} to authenticated;` を別途実行
- **注**: Supabase Table Editor (GUI) で作成した場合は自動 GRANT される。SQL Editor 限定の落とし穴
- **既存解**: `docs/migrations/2026-05-05_tags_x1_grants.sql` 参照

### 5-3. AMZN の FMP image-stock は欠陥（識別困難）

- **症状**: FMP の AMZN ロゴは「矢印のみ + 上部空白」で 24px 円形だと何も見えない
- **対応**: TradingView の正式ロゴを使用（tickerSlugs.js で `AMZN: 'amazon'` 登録）
- **教訓**: ロゴ表示は必ず複数銘柄（特にメガキャップ）で実機確認すること

### 5-4. Railway の deployment ID と git commit hash は別物

- **症状**: ユーザーが Railway dashboard の `cd6e5c24` を git hash と誤認
- **真実**: Railway dashboard の 12 桁 ID は **deployment ID**（その回のビルド・デプロイ操作の識別子）。git commit hash とは別物
- **確認方法**: Deploy Logs の最初の方に「Building from commit XXXXXXX」の行 → そこの 7 桁が git commit hash
- **判定**: `Active` 表示 = 現在動いているデプロイ。バンドルハッシュ変化 + 新コード文字列 grep で実反映を確認可能
