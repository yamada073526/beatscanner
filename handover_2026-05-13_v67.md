## 4. 触ると危険な箇所 (CLAUDE.md + v66/v67 追加)

### v67 追加
- **`animation-fill-mode: both` + transform の罠** (詳細 `feedback_press_feedback_delta.md`):
  - running animation の forwards fill 由来 transform は **`:hover` / `:active` の transform を CSS spec 上完全に上書き** (animation origin > author origin、specificity / !important 無効)
  - 新規 card 系 (`:hover` / `:active` で transform 使う) では `getAnimations()` で animation 確認 + `animation-fill-mode` を `backwards` か `none` に
  - pixel が動かない疑いがあれば DevTools Computed の `transform` 行で matrix を読む (Style pane の overridden 表示は misleading)
- **Visual Harness Exception** (CLAUDE.md 配下):
  - `frontend/scripts/snap-*.mjs` の headless Playwright (60s 内 + 自動 teardown) は preview ban の限定例外
  - long-running dev server / preview / HMR / `localhost:5173` bind / user セッション並走起動は引き続き禁止
- **Sentry tunnel ハマりポイント** (`sentry_setup.md` 参照):
  - tunnel から Sentry へ forward する URL には `?sentry_version=7&sentry_key=<public_key>` が必須
  - public_key は DSN の userinfo 部 (`<key>@host` の前半)、`urlparse(dsn).username` で取れる
  - direct send は SDK が自動付与するが tunnel は backend 側で明示が必要
