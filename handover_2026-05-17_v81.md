## 3. 触ると危険な箇所 (v81 新規 + 継承)

### v81 新規確定原則

**HMAC unsubscribe token は CRON_SECRET 流用 (handover v81 §0-B)**:
- 新 env var 追加せず、 `os.environ.get("CRON_SECRET", "")` を `hmac.new(secret, user_id.encode(), hashlib.sha256)` のキーとして再利用
- secret 未設定環境 (local dev 等) では token 空文字、 verify 全 false で reject (fail-safe)
- `CRON_SECRET` を切り替えると **既発行の List-Unsubscribe URL が全て無効化**される (本番運用では rotation 注意)
- 将来 high-volume 配信になったら `UNSUBSCRIBE_TOKEN_SECRET` を分離するのが推奨 (key rotation 影響を unsubscribe URL のみに限定)

**Pro 文言 Trust Cliff (handover v81 §0-E、 next session §2-B fix)**:
- Cup overlay の locked banner が「Pro で全データ解放」 と表記、 実装は Premium のみ
- Pro subscriber が混乱する可能性、 文言を Premium 明示 (or Pro/Premium 階層明示) に修正必須
- 同様の「Pro で 〜」 表記は他 component にも残存する可能性 → grep でアドホック検索

**CustomScreenerPanel は SPA-only (handover v81 §0-G)**:
- `?layout=classic` で旧 UI 強制、 mobile も SPA、 PC default の `?layout=workspace` には未マウント
- 「Cup-Handle scanner chip を使いたい」 user は明示的に SPA を開く必要
- 新 UI port は §2-C のタスク、 完了後はこの限定が解消

### v80 から継続 (handover v80 §3 参照、 削除可)
