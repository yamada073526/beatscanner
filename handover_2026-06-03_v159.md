### backend: `GET /api/screener/universe-meta` (純データ・LLM 非経由・§38 risk なし)

- 本番検証: `count=2449` / `mega 897・mid 949・small 603` / top sectors Tech405・Healthcare373・Financial355・Industrials332・ConsumerCyclical255・Energy168 / null sector 2件のみ / AAPL=Tech/mega・XOM=Energy/mega 正答。

### frontend: RsScannerResults に折りたたみ filter panel (`CustomScreenerPanel.jsx`)
- 折りたたみ (SlidersHorizontal + 「セクター・時価総額で絞り込み」 + active数 badge + ChevronDown、 default 閉)。
- **sector chip**: 結果に存在する sector を count 集計 → 実セクター **top6 data-driven + その他** に畳む (固定5リストが GOOGL/AMZN を埋もれさせる問題を回避)。 各 chip 「ラベル + count」。
- **mcap帯 chip**: 大型/中型/小型、 各 count + 数値 tooltip + 「(基準日) 時点」表記。 該当0件帯は非活性 dim。
- 複数選択 (sector AND mcap、 各次元内 OR)、 sort行右に「N/M 件」、 0件時「選択した条件に一致する銘柄がありません」。
- **module-scope cache** (`loadUniverseMeta`) で remount 跨ぎ共有 (DiagramCard remount 教訓)。 tab 切替で RsScannerResults unmount → filter state 自動 reset。
- 全 chip は Chip primitive (active=accent ブランド色=中立操作)。 **design-system-check PASS** (raw hex/!important/発光/chip 違反なし)。
