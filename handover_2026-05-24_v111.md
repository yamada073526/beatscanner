## 2. v111 polish 詳細 (UI/UX 1 体合議 verdict)

user dogfood (v110 KpiStrip 拡張後) feedback 2 件:

### 議題 1: 「Buyback比率」 だけ改行発生 → label 短縮 + grid 微調整
- **fix**: 「Buyback比率」 (英 8 文字) → **「自社株買い」** (日 5 文字、 視覚幅 30% 短縮)
- Bloomberg JP / 日経の標準語、 5 原則「2 秒理解」 整合
- grid: `repeat(auto-fit, minmax(140px, 1fr))` → `minmax(130px, 1fr)` で 7 chip × 130 = 910px、 ラップトップ幅 (1280px) で 1 行収容
- 却下: 「BB比率」「還元比率」 略語 (認知過多) / `repeat(7, 1fr)` 固定 (chip 増加耐性なし)

### 議題 2: count-up animation 全 chip 拡張
- 以前の sub-agent verdict「数字 表示時 count-up が点数アップ」 反映
- parseKpiValue に **`plainDecimalMatch`** `/^(\d+(?:\.\d+)?)$/` 追加 → Forward P/E (17.4) / PEG (0.30) を count-up 対象化
- 配当性向 (0.60%) / 自社株買い (98.00%) は **既存 `pctMatch` で既に対応済** (suffix '%' で match)
- displayValue format: 原値の小数桁を保持 (`originalDecimals` 動的判定で 17.4 → 1 桁 / 0.30 → 2 桁 / 98.00 → 2 桁、 旧 `toFixed(2)` 一律から改善)
- 「4/5」 (条件合致) と「—」 (EPS BEAT) は count-up 対象外維持 (中間値が無意味)
- 適用範囲: 全 7 chip → 全数値 chip に count-up 統一、 Aman 級「驚き / 興奮」 演出 + 「未ロード」 体感解消

## 4. 触ると危険な箇所 (v110 + v111 新規教訓)

### v110 教訓
- **valuation-extras endpoint は LLM 一切介在せず純数値層**: aggregator/ 配下に置かない、 visualizer/ も呼ばない、 prompt.py に数値計算指示を書かない (Hallucination Guard 4 重防御 pre-commit hook で BLOCK)
- **配当 chip は事実数値のみ + trend: 'neutral' 固定**: 金商法 §38 (断定的判断提供禁止) 回避、 amber chip / verdict warning は禁止
- **KpiStrip auto-fit grid は chip 増加で改行 risk**: 8 chip 以上は grid 設計再検討必要

### v111 教訓
- **count-up 表示桁は原値の小数桁を動的保持**: `toFixed(2)` 一律だと 17.4 → 17.40 で表示 mismatch、 `originalDecimals` 判定で原表示維持
- **chip label は日本語短縮優先 (Bloomberg JP / 日経 idiom)**: 英 8 文字「Buyback比率」 → 日 5 文字「自社株買い」 で視覚幅 30% 短縮
- **grid minmax 130px 安全境界**: 140 → 130 で 7 chip × 130 = 910px、 一般 ラップトップ (1280px) 1 行収容、 mobile (375px) なら 2 列折返し維持

### 既存教訓 (v107 + v108 から継承)
- early return より前に全 hooks (Rules of Hooks)
- ChapterTabs primitive SRP 維持 + triggerOnMount prop pattern
- halo 不発時は early return + data ready の有無を確認、 useEffect deps に `data` 追加
- headerCollapsed state は keep + component 側で false 固定
- 金融 chip は事実数値のみ (金商法 §38 / 景表法 §5)
