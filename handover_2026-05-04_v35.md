## 2. Task B: LP 二重ヒーロー問題修正

### 問題
v34 で LP を組み込んだ際、未ログイン時に **既存ホームのヒーロー** と **LP の Hero** が両方表示されていた:
- 「決算を、瞬時に」の文字が画面内に **2 回出現**
- ApiKeyBanner、MarketWidget も重複表示

### 原因
v34 で以下 4 要素に `&& !showLP` を gate していなかった:
- [App.jsx:673](frontend/src/App.jsx) `ApiKeyBanner`
- [App.jsx:679](frontend/src/App.jsx) `MarketWidget`
- [App.jsx:681](frontend/src/App.jsx) 旧 Hero ブロック ← 主因
- [App.jsx:931](frontend/src/App.jsx) `GuidanceCard` (edge case)

### 修正
全て `{!showLP && ...}` で gate。LP 表示中は LandingPage コンポーネントだけがレンダリングされる状態に。

**コミット**: [`c3781bd`](https://github.com/yamada073526/beatscanner/commit/c3781bd)

## 3. Task C: LP デザイン統一

### 背景
v34 で実装した LP は独自スタイル（`fontSize: 'clamp(...)'`, 独自 `borderRadius`, `padding` 等）が多く、既存ホーム画面のデザインと不一致だった。

### 統一内容

#### フォント
- 独自の `font-family` / `font-size` / `font-weight` 上書きを削除
- 既存 `.hero-title` / `.hero-badge` / `.section-heading` / `.section-subtext` クラスに委譲

#### カラー
- 全テキストを CSS 変数化: `var(--text-primary/secondary/muted)`
- 全背景を CSS 変数化: `var(--bg-primary/card/subtle)`
- ボーダーを `var(--border)` に統一

#### ボタン
- `PrimaryCTA`: シアン塗り (`#22d3ee`, hover `#06b6d4`) — Hero/Footer の主役 CTA
- `OutlinedCTA`: 既存 `.cta-btn` クラスを継承（rgba 背景 + シアン枠 + glow hover）

#### セクション見出し
- `SectionLabel`: `section-subtext` + uppercase + シアン色（小さい上ラベル）
- `SectionTitle`: 既存 `.section-heading` クラス（18px / weight 500）
- 独自の `clamp()` フォントサイズ指定を全削除

#### カード
- 機能カード・料金カードを `.panel-card` クラスに置き換え
  → 既存のホバー演出（`translateY(-5px) + cyan glow + ring`）が自動適用
- 角丸 12px / `border 1px var(--border)` / 背景 `var(--bg-card)` で統一
- Pro プランのみ強調用に `border rgba(34,211,238,0.55) + glow`

### 効果
- ダーク/ライトテーマ自動対応
- 既存ホーム画面とシームレスな視覚的一貫性
- 独自スタイル 200 行以上削減 → メンテナンス性向上

**コミット**: [`f75a09e`](https://github.com/yamada073526/beatscanner/commit/f75a09e)

## 4. Task D: LP CVR 改善 Phase 1（5 点）

### Fix 1: 社会的証明バーを Hero CTA 上に追加

```
📊 対応銘柄数: 10,000銘柄+    ⚡ 5条件を自動判定    🔄 毎朝4時に自動更新
```

`text-secondary` 色で控えめながら、3 つの定量的シグナルで初見ユーザーの信頼感を確立。

### Fix 2: クレカ不要バッジを CTA 直下 3 個に分割

| 旧 | 新 |
|---|---|
| 単一行「クレカ不要・3銘柄/日まで無料分析」 | `✓ クレカ不要` `✓ 3銘柄/日まで永久無料` `✓ 30秒で登録完了` の 3 バッジ |

「3 つの安心感」を視覚的に与えて登録障壁を最小化。

### Fix 3: 年払いバッジ強化

| 旧 | 新 |
|---|---|
| section-subtext 「年払いなら ¥9,800（2ヶ月分お得）」 | シアン pill 「🏷️ 年払いで2ヶ月分お得（¥1,960節約）」 |

節約金額を**円**で明示することで年払い動機を強化。

### Fix 4: FAQ を 3 問 → 6 問に拡充

#### 追加した 3 問
1. **Q. 投資初心者でも使えますか？**
   → 専門知識不要・5条件と AI 解説でシンプルに
2. **Q. 日本語で使えますか？**
   → 完全日本語対応・自動翻訳機能搭載
3. **Q. 決算結果はいつ反映されますか？**
   → 数時間以内に自動更新・市場の声は毎朝4時

#### UX 改善
- `FAQItem` に `defaultOpen` prop を追加
- **1 問目「Free/Pro 違い」をデフォルト展開状態**に → 訪問者が一度もクリックせずに最重要 FAQ を読める

### Fix 5: キャッチコピー・サブコピー・β バッジ変更

| 要素 | 旧 | 新 |
|---|---|---|
| **タイトル** | 決算を、瞬時に読み解く。 | **決算 Beat/Miss を、5秒で判定。** |
| **サブコピー** | 売上・EPS・バリュエーションをAIが図解。Beat/Miss・ブル/ベアを即判定。 | **もう夜中に決算資料を読まなくていい。5条件を自動チェックして、買い継続か撤退かをデータで判断。** |
| **β バッジ** | β版・先着ユーザー募集中 | **β版・招待制プレビュー公開中** |

#### 設計の意図
- **「5秒」という具体数値**で価値訴求の鋭さ向上（マーケティング業界の黄金パターン）
- サブコピーを**機能説明 → ベネフィット訴求**へ転換（「夜中に読まなくていい」で読者の痛みを直接刺激）
- β バッジを**「招待制」**に変えて希少性を演出 → FOMO 効果

**コミット**: [`b59ddaa`](https://github.com/yamada073526/beatscanner/commit/b59ddaa)

## 8. v35 で確立した永続ルール候補

CLAUDE.md への昇格を検討:

### 8-1. LP / 説得型ページの設計原則
- **Hero CTA の上下に社会的証明 + 安心シグナル** を配置
- 数値はできるだけ**具体化**（「お得」より「¥1,960 節約」）
- β バッジは「先着」より「**招待制**」の方が希少性訴求に効く

### 8-2. デザイン統一の指針
- 新規 UI を作る際は、**まず既存 CSS クラス（`.hero-badge`, `.panel-card`, `.cta-btn`, `.section-heading`, `.section-subtext`）が使えないか確認**
- 全テキスト・背景は **CSS 変数経由**（`var(--text-primary/secondary/muted)` / `var(--bg-card)` / `var(--border)`）
- 独自 `fontSize` リテラルは原則禁止 → セマンティッククラスを使う

### 8-3. Stripe Hosted Checkout の運用
- フロントに `pk_*` を直接埋め込まない（Stripe.js 不使用）
- バックエンドの `STRIPE_SECRET_KEY` のみが本番/テスト判定
- `VITE_STRIPE_PUBLISHABLE_KEY` は将来 Stripe Elements を入れる場合の保険として保持
