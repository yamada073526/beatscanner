## 🔴 朝の最優先 — 判断キュー (ship 前に user gate 要・推奨付き)

### ★ 最重要: 戦略タイルが半幅ペインで激しくラベル折返し (洗練さ後退の核と推定)
- **実測** (snap-strategy-bar-width.mjs): スクリーナーペイン = viewport の約50%幅。
  - 1280px → tile 136px → 全4ラベル折返し (「セクター別リーダー」は3行)
  - 1600px → tile 183px → 2ラベル折返し / 1920px → tile 224px → 折返し無し
- **真因**: `.screener-strategy-bar` の grid は `repeat(4,1fr)` + **viewport 基準** media query (680/420px)。半幅ペインに置かれても viewport が広いと4列のまま → ノート PC 幅で cramping。CSS drift でなく embedded layout-context 問題。
- **選択肢** (要 user 判断): (A) container query で pane 幅基準にし狭い時 2列化 (推奨・robust) / (B) スクリーナーペイン自体を幅広化 (IA 変更) / (C) ラベル font-size 微縮 + nowrap。
- 証拠 PNG: `frontend/.visual/vd-prod-preset.png` / `vd-prod-custom.png`。
