## 3. 触ると危険な箇所 (v73 新規 + v72 継承)

### v73 新規確定原則

**lucide-react は React.forwardRef** → `typeof Icon === 'function'` 判定で落ちる罠
- `frontend/src/components/LandingPage.jsx` FeatureCard で発生、 LP 真っ白 emergency
- 正しい判定: `Icon ? <Icon /> : null` の truthy 判定
- `feedback_lucide_forwardRef.md` memory anchor 候補

**SVG `<title>` native tooltip は OS default で luxury 崩壊**
- BacktestPage の TradeBarChart / YearBarChart / SectorBarChart で同根
- 解決: React state + 外側 div の Aman 級 popover (240px rgba(15,23,42,0.92) + backdrop-blur)
- TickerBadge は logo + ticker 両方内包 (`TickerBadge.jsx:67-70`)、 追加 span は二重表示

**PDF と Web 版の chart metric は完全一致必須 (Trust Cliff)**
- 業界比較 bar の順序 (低→高昇順) + metric (勝率) を統一
- 旧版で SPY「持つだけ」 を passive return % で表示していたが、 Web 版は「勝率」 → 別 metric だった
- `_run_jijima5_backtest` の `kpis.win_vs_spy_rate_pct` を使用

**PDF Visual-first 規律 (subagent レビュー教訓)**
- 「Bloomberg/Morningstar 級信頼性訴求」 は **作り手の動機**で、 ユーザー動機ではない
- PDF の北極星は「**この PDF を見せられた友人が BeatScanner.app を開く**」 = 広告物
- 文字壁回避: 大数字 1 + 大 chart 1 + 短文 caption、 白比率 45%
- 5 条件は icon + 短文 (PIL primitive で Lucide 風 cyan icon)、 SPIVA は horizontal bar 3 本 (Web と同 pattern)
- footer は **URL のみ**、 「Aman 級」 等の自画自賛は信頼下げる
- `feedback_pdf_visual_first.md` memory anchor 候補

**ダークラグジュアリーは半分しか真ではない**
- Aman / Ritz-Carlton 公式 web は実は **light 寄り** (white background + 大判 photo)
- BeatScanner で dark default 採用は **数字密度 (Bloomberg/TradingView/Webull 慣習)** が主要根拠
- 「dark = luxury」 は web では少数派、 fashion luxury (Saint Laurent / Cartier) で成立
- `feedback_dark_luxury_nuance.md` memory anchor 候補
