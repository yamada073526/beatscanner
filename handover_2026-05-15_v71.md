## 3. 触ると危険な箇所 (v71 で確定した永続原則)

### 新規追加 (詳細は anchor 参照)

**Pane 3 抽象化原則** → `memory/feedback_pane3_detail_view.md` (旧 feedback_indices_view_design.md を拡張改名)
- selectedTarget discriminated union `{type, id}` が SSOT
- URL prefix `idx/pf/t/judg` は **freeze**、 'チャート' key と同列の不変
- 新規 detail target 追加時は本 anchor の「How to apply」に従う

**Phase 2.1 で確定した 5 永続原則** → 上記 anchor の「Phase 2.1 改善履歴」
1. cross-state 同期は **setter middleware**、 useEffect は anti-pattern (cascade re-render)
2. 口座名は **`useAccountName(id)` hook 経由のみ** (prop drilling 禁止)
3. chart latency は **mount 時の他期間 fire-and-forget prefetch**、 SWR / TanStack Query 不要
4. chip 意味は **visible caption + aria-label** (title 属性は mobile / NVDA で読まれない)
5. 金融用語 (α / Alpha) は **記号サフィックス + tooltip** で retail に学習させる

### v70 から継続 (handover v70 §5 参照、 削除可)
- Modified Dietz `start_shares = period_from - 1day` (二重控除回避)
- Chip primitive 規律 (inline style 禁止、 variant prop でのみ宣言)
- ChipGroup.Separator 等の compound pattern は Vite minify 不安定 → named export
