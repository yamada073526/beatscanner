## 🟡 重要知見 (永続化済、 memory anchor 候補)

### `_get_spy_history()` は dict 返却
新規 endpoint で SPY closes 使うときは `(spy_history or {}).get("closes") or []` で取得。 5-tuple list ではない。 私が cron_rs_scan 初回実装で間違えた (commit 86e55ed で hotfix)。

### `_fetch_ohlcv_3y(ticker)` は 5-tuple `(times, highs, lows, closes, volumes)` を返す
これは `cron_cup_scan` で正しく unpacking されている。 RS scan でも同様に流用。

### FMP `^NDX` は Premium plan でも取得不可 (special endpoint)
Active / Premium plan ともに 402 Premium Query Parameter error。 Ultimate plan のみアクセス可。 → **^IXIC (NASDAQ Composite、 ~3000 銘柄連動)** で代替推奨。

### GitHub Actions schedule は遅延発火が頻発
v113 (19:05 UTC) / v118 (19:35 UTC) は schedule で動いていたが、 5/27 朝は **20:57 UTC / 21:15 UTC** に遅延発火 (user 5 時帰宅時点でまだ未動作 → 6 時頃発火)。 Morning Health Check (19:00 UTC) は時間通り。 GitHub Actions 高負荷 skip / delay の典型。 → schedule cron は 「いつ動くか不確実」 という前提で運用、 dogfood は遅延考慮で 06:30 JST 以降推奨。

### Anthropic auto cache (同一 system 5 分 idle) は ClaudeClient.complete() でも有効
multi-block cache_control: ephemeral でなくても、 same system 文字列なら 5 分以内に呼ぶと cache hit。 main.py の SUMMARY_SYSTEM_PROMPT に文体憲法 inline append でも cost 効率 OK。
