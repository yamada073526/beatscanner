## 4. 触ると危険な箇所 (v103 + v104 教訓)

### v103 から継承
- sticky 検索バー / 発光系 / VITE_ARG/ENV / Hallucination Guard / FMP /stable
- ReadingMode 全面 Overlay (Linear/Gmail 流)
- Pane 3 章 2 タブ切替 (isV3) で章 3 QuarterlyHistoryTable は二重表示防止 skip
- NewsItem `ws-pane4-news-item` CSS v66 backwards 戦略
- Pane4Inspector orchestrator 化後の hook dependency chain

### v104 新規教訓
- **isPane3V3() feature flag は default OFF**。 dogfood feedback まで強制反映不可。 ?pane3_v3=1 試用 → user dogfood OK → default ON 昇格は別 commit
- **章 2 ChapterTabs と章 3 QuarterlyHistoryTable の二重表示問題**: isV3 ON 時に章 3 を必ず `!isV3 &&` で gate。 isV3 default ON 化時に章 3 から QuarterlyHistoryTable 永久削除 (orphan code 撤去)
- **JSON-LD は AI 検索 (ChatGPT / Perplexity) 経由集客で効果**。 効果は 1-2 ヶ月後の Sentry referrer tag で計測、 早期に lots of 構造化はしない
- **llms.txt 拡充は Premium 価格を含めない**: ¥1,800/月 vs Pro ¥2,000/月 の価格逆転問題が release 後再考のため、 Pro plan のみ訴求
- **SEC EDGAR 直 fetch は User-Agent 必須**: `"BeatScanner support@beatscanner.example"` で安全、 削除すると 403 で blocked
- **App.jsx は lucide-react 未 import**: emoji を React component に置換するには import 追加が必要、 1 文字 emoji 置換は string 置換より大型変更 (spawn_task chip 推奨)
- **ChapterTabs の active tab 内容のみ render**: lazy mount で perf 最適化。 ただし fetchQuarterlyHistory 等の effect は QuarterlyHistoryTable mount 時のみ trigger、 active tab 切替時に fetch + cache 30 分 hit が綺麗に成立

## 5. 永続 memory anchor 候補 (v104 新規)

- **feedback_chapter_tab_pattern.md** (新規): 章内 tab interface を ChapterTabs primitive で実現する pattern。 segmented tab CSS 流用 + ARIA role="tablist" + keyboard nav + active tab lazy mount。 default OFF feature flag (URL + localStorage) で dogfood 安全
- **feedback_seo_jsonld_pattern.md** (新規): index.html JSON-LD で WebApplication + Offer + Organization schema 埋込。 AI 検索引用最適化、 1-2 ヶ月後の referrer tag で計測
- **feedback_sec_edgar_direct_fetch.md** (新規): SEC EDGAR submissions.json を generic helper (`_fetch_filings_from_sec_edgar(sym, form_type, limit)`) で 8-K / 10-K / 10-Q 共用化。 User-Agent 必須、 12h cache、 FMP non-dependent で free user 開放可能
