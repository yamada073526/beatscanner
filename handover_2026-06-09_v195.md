## 🟢 C-3 keep-mounted (B案、 user 選択) の設計

「競合へ遷移する前の状態へロードなしで一瞬で戻りたい」。判定 result は `detailFor`(resultCacheRef) を props で同期受け取りするため瞬時だが、**panel (chart/会社概要/ガイダンス/analyst/news/IR) は自前 async fetch のため ticker 変化で loading state にリセット**される構造が「リロード」の正体。

- **`DetailStack.jsx`** (新規): `detailHistory` の各 ticker ごとに `<JudgmentDetail tickerOverride={t}>` を mount し続け、active な1つだけ `visibility:visible`、他は `visibility:hidden` + `inert`。各 instance は `position:absolute; inset:0; overflowY:auto`。**visibility:hidden は display:none と違い layout 維持で scrollTop が DOM 保持**される。DetailStack root が height:100% で PaneContainer を埋めるため PaneContainer 自体は scroll せず各 instance が内部 scroll を供給。
- **`JudgmentDetail.jsx`**: `tickerOverride` prop 追加 (`undefined`=context / 明示値=固定)。子 component は誰も `useJudgment` を直読みせず `ticker={selectedTicker}` props で受けるため **L322 の 1 点 override で全 panel に波及**。
- **適用は normal(home) detail path のみ** (Workspace.jsx L1122)。screener/indices/SPA は単一 JudgmentDetail のまま (blast radius 限定)。
- **2体合議 (frontend + qa) の指摘 3 件を修正 (`ec6934c`)**:
  1. focus漏れ → hidden instance に `inert` (WCAG 2.1.1)
  2. 重複id scroll (acc-header-sec-* が複数 instance で重複) → FiveConditionsCard が `[data-detail-instance][inert]` 配下を除外し active instance の header を選ぶ
  3. cross-tab breadcrumb 汚染 (Trust Cliff) → `resetDetailHistoryToCurrent` を tab 切替 effect で呼び現在地のみに reset
- **authed harness 検証 PASS** (`snap-c3-keepmount.mjs`): MSFT 遷移後 instance=2 (AAPL inert/hidden で charts=12/panels=7 保持) / back-nav 300ms で AAPL loading=0/marker 永続 = remount/再fetch ゼロ / layout 崩れなし / pageError 0。

## ⚠️ 危険箇所 / 運用知見
- **keep-mounted の blast radius**: DetailStack は home path のみ。`setActiveTicker` 18 callsite + `detailHistory`(computeNextDetailHistory) は不変。rollback は Workspace.jsx L1122 を `<DetailStack>`→`<JudgmentDetail>` に戻す 1 箇所 + tickerOverride は undefined default で無害。
- **重複 id 残**: keep-mounted で `acc-header-sec-*` / `sec-*` id が複数 instance で重複する。getElementById 系を新規に書く際は `[data-detail-instance][inert]` 除外 or instance scope を徹底 (FiveConditionsCard が前例)。
- **glow 系**: `useArrivalSpotlight` は hidden instance の card も is-arriving を付与するが visibility:hidden で不可視・active card は Set 方式で常に正しく glow するため**実害なし** (検証済、hook 非改変)。`.panel-card/.surface-card` / AnalystTargetCard / sticky 検索バー / VITE_ ARG/ENV / aggregator LLM import は不触。
- **expandedSections 越境** (既存・軽微): condition-click の force-open が nav clear なしで全 instance 共有。単一 instance 時代からの挙動で keep-mounted の新規 regression ではない。
- **authed harness が使える**: `frontend/.env` に DOGFOOD creds 配置済 (gitignore)。`node --env-file=.env scripts/snap-*.mjs` で Premium 認証注入 → authed chart も headless 視覚検証可能 (旧「authed chart は headless 検証不可」制約は解消)。
