## 1. Task 1: Stripe 本番モード切替（コード監査結果）

### 監査結果
[backend/app/main.py](backend/app/main.py) の Stripe 実装は **完全に環境変数依存**:

| 用途 | 取得方法 | ハードコーディング |
|---|---|---|
| Stripe API キー | `os.environ.get("STRIPE_SECRET_KEY", "")` | なし |
| Webhook 署名 | `os.environ.get("STRIPE_WEBHOOK_SECRET", "")` | なし |
| 月払い price ID | `os.environ.get("STRIPE_MONTHLY_PRICE_ID")` | なし |
| 年払い price ID | `os.environ.get("STRIPE_YEARLY_PRICE_ID")` | なし |
| Publishable key | `import.meta.env.VITE_STRIPE_PUBLISHABLE_KEY` (フロント) | なし |

フロントエンドにも `sk_test_*` / `pk_live_*` 等のハードコーディングは無し。

### 結論
**Railway 環境変数を Live キーに差し替えるだけで本番モードに切替可能**。コード変更は一切不要。

### ユーザー実施手順（次セッションで案内）

   - 月払い ¥980/月 → 新しい `price_xxx` をメモ

#### ② 本番 Webhook を作成
1. Developers → Webhooks → Add endpoint

3. リッスンするイベント:
   - `checkout.session.completed`
   - `customer.subscription.updated`
   - `customer.subscription.deleted`
4. 表示される `whsec_xxx` をメモ

#### ③ Railway 環境変数を Live キーに更新
以下 5 つを差し替え（テスト用キーを上書き）:
```

STRIPE_WEBHOOK_SECRET        = whsec_xxx       (旧: whsec_*テスト用)
```

```bash

```

## 2. Task 2: 未ログイン LP（ランディングページ）新設

### 表示条件
[App.jsx:178](frontend/src/App.jsx) で派生値として定義:
```javascript
const showLP = activeTab === 'home' && !result && !user && !loading;
```

| 状態 | LP 表示 |
|---|---|
| 未ログイン・ホームタブ・分析結果なし | ✅ 表示 |
| ログイン済み（user 確定） | ❌ 非表示（既存ホーム画面） |
| 分析結果あり (`result` 確定) | ❌ 非表示 |
| 分析中 (`loading=true`) | ❌ 非表示（チラつき防止） |
| 別タブ（判定/決算/チャート） | ❌ 非表示 |

### Pro CTA → ログイン後自動チェックアウトの仕組み

未ログイン状態で `startCheckout('monthly')` を呼んでも、useSubscription 内の `if (!supabase || !user) return;` で何も起こらない。これを解決するため:

1. LP の「7日間無料で試す」クリック
   ```javascript
   localStorage.setItem('bs_post_login_intent', 'checkout_monthly');
   signInWithGoogle();  // Google OAuth 開始
   ```

2. App.jsx の useEffect が `user` 確定を検知
   ```javascript
   useEffect(() => {
     if (!user) return;
     const intent = localStorage.getItem('bs_post_login_intent');
     if (intent === 'checkout_monthly') {
       localStorage.removeItem('bs_post_login_intent');
       setTimeout(() => startCheckout('monthly'), 100);
     }
   }, [user, startCheckout]);
   ```

3. `setTimeout(100ms)` で React の状態更新後に startCheckout 呼び出し → Stripe Checkout 画面へリダイレクト

### デザイン統一の工夫

- **共通スタイルトークン** (`C` オブジェクト) を LandingPage.jsx 内で定義してシアン色を一元化
- **`PrimaryCTA` / `SecondaryCTA` コンポーネント** を内部定義し、ボタンスタイルを統一
- **`clamp()` でフォントサイズ可変** (`clamp(32px, 6vw, 52px)`) → モバイル/PC でスムーズに変化
- **Hero に NVDA モックアップ** を CSS のみで描画（追加画像不要）
- **全幅ヒーロー** のため `width: 100vw` + `marginLeft: calc(-50vw + 50%)` で親 `max-w-6xl` を脱出

## 6. v34 で確立した永続ルール候補

CLAUDE.md への昇格を検討:

### 6-1. ログイン誘導後の意図維持
- 未ログインユーザーが特定機能（Pro 等）を選んだ場合、**意図フラグを localStorage に保存** してログイン後に自動継続
- パターン: `localStorage.setItem('bs_post_login_intent', '<action>')` → useEffect で `user` を watch → 該当アクション実行
- 同じパターンで「市場の声フル表示」「ウォッチリスト追加」等にも応用可能

### 6-2. LP のような全幅セクション
- 親の `max-w-6xl` を脱出するには `width: 100vw; marginLeft: calc(-50vw + 50%); marginRight: calc(-50vw + 50%);`
- このパターンは sticky 検索バーでも使用済み（CLAUDE.md にも記載あり）

### 6-3. 環境変数差し替えのみで本番/テスト切替
- Stripe 等の決済 SDK を実装する際は、**コード内に test/live を一切書かない**
- 全て環境変数経由で取得（`os.environ.get`, `import.meta.env`）
- これにより本番化時のリスクを最小化
