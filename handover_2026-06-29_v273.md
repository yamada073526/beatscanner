## 🟡 重要知見 (このセッション固有・永続化候補)

- **ground-truth で handover 前提を 2 点修正** (main.py 実コード読み):
  - `op_cf_margin` は **ratio(0–1)** 格納 → 表示は `×100` (`QuarterlyHistoryTable` と同スケール)。
  - `ocf_gt_netincome` は `canslim/rows` 応答に**無い** (DB列はあるが SELECT 未含)。代わりに quarterly-history の `cfps_actual/eps_actual` から OCF÷純利益比率を frontend 算出 → backend 追加不要・4a cache_key 衝突回避。
- **quarterly-history cache_key は `v3`** (main.py:6922)。4a (PR #117) の `v4` は別ライン。§② build-out は v3 のまま endpoint 未改変。
- **IA 教訓 (multi-review 2 連発の学び)**: 「常時図解が善」は理論。実機では header 1行に 76px sparkline を詰めると (各棒~5px) hover 不能 + 過密。**dogfooder 洞察「人力の代替の価値は 8Q を自動集計して方向ラベルを付けること自体であり、グラフ常時表示ではない」** → header に方向 chip を残し精密 sparkline は fold へ、が実使用最適。Fitts則: hover には各棒12px+ = フル幅200px+ 必要。
- **canslim/rows 配線**: `fetchCanslimRows([ticker])` を判定タブに初配線 (EPS CAGR 3年用)。rows は ticker 大文字 key、case-insensitive lookup 実装済。nightly cron 未計算銘柄は null→pillbox 非表示。
- §② chip は §①`EarningsGrowthSpark` と同じ inline span idiom (`.ds-chip` 不使用・token+color-mix)。Chip primitive は mockup の filled chip と display variant が不一致のため非採用。
