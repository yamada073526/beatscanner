## 🟡 重要知見 (永続化候補、 本 session 教訓)

### article_pipeline は claude-sonnet-4-5 一択 (v115 r1 確立)
- Sonnet 4.6 (`claude-sonnet-4-6`) は **prefill 非対応** → JSON-only 強制に必須の `prefill="{"` で 400 error
- Opus 4.7 (`claude-opus-4-7`) は **temperature 非対応** + prefill 非対応
- backend 他 7 箇所で稼働実証の `claude-sonnet-4-5` のみが prefill + temperature + cache_control 全 OK
- memory anchor `feedback_cost_efficient_operation.md` の「sub-agent default Sonnet 4.6/4.7」 と矛盾する例外、 article_pipeline 内 narration 層は実装側制約優先

### Starlette `StaticFiles(html=True)` は nested directory の index 自動 serve しない (v113 P3 確立)
- `dist/index.html` (root) は OK、 `dist/articles/<slug>/index.html` (nested) は html=True でも 404
- 対処: FastAPI `@app.get("/articles/{slug}")` で 5 line bridge (commit `006f3be`)

### EarningsRing 真因 = 外側 conditional wrapper (v115 r2 教訓)
- EarningsRing.jsx 内で `state='unknown'` を扱う設計にしても、 親 Hero.jsx で `Number.isFinite(nextEarningsDays) && <EarningsRing>` が gate していたら無意味
- **fallback UI を component 内で実装する場合、 親 conditional render を必ず確認**

### FactChecker strict 度 sentence_type pre-classify pattern (v115 r1 教訓)
- 旧: sentence ≒ expected_fact 整合判定 → partial coverage + speculative wrapper が 90%+ mismatch
- 新: Python で sentence_type を 'hard_fact' / 'interpretive' 分類 → interpretive は確率 wrapper + partial coverage 許容
- 実 launch 効果は **明朝 04:00 JST cron 自動発火後** に判定

### Railway service vars は Dockerfile ARG 必須 (v113 P3 + v115 r1 教訓)
- `VITE_*` 変数 + `SUPABASE_SERVICE_ROLE_KEY` 等 build-time に必要な変数は Dockerfile に `ARG` + `ENV` 連鎖必須
- Service Variables だけでは build stage に届かない

### Railway build hash ≠ local build hash (v113 P3 教訓)
- Local `npm run build` produces `index-DHBuafZY.js`、 Railway は `index-2KNAdu0f.js` (Node ver / npm install state の deterministic 差)
- Local hash で poll する deploy 完了検知 pattern は壊れる → endpoint availability 検知 (新 endpoint の 200) を使う

### Insider/10-K デザイン: IR Links と class 共有で hover 統一 (v115 r3 教訓)
- 個別 component の inline style hover (onMouseEnter / Leave) は specificity 低くて override 不可
- IR Links の `.ir-link-item` class (border-left slide-in + ↗ translate) を流用すれば一貫性 + メンテ性 両得

### ticker internal link は財務略語の網羅的 blocklist 必須 (v115 r1 教訓)
- PEG / CFPS / EPS / EBITDA / ROIC / WACC / CAGR / QoQ / NPV / IRR / GAAP / IFRS / ESG / SaaS / FY24-27 等
- 初期実装で 「IT/AI/SDK」 等は入れたが財務略語は抜け落ちがち、 user dogfood で発覚
