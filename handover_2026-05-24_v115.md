## 🟡 重要知見 (永続化候補、 本 session 教訓)

### model 選択: article_pipeline は claude-sonnet-4-5 一択
- Sonnet 4.6 (`claude-sonnet-4-6`) は **prefill 非対応** → JSON-only 強制に必須の `prefill="{"` で 400 error
- Opus 4.7 (`claude-opus-4-7`) は **temperature 非対応** + prefill 非対応 → article_pipeline で使えない
- backend 他箇所 7 件で稼働実証の `claude-sonnet-4-5` のみが prefill + temperature + cache_control 全て互換
- 既存 memory anchor [[cost-efficient-operation]] の「sub-agent default Sonnet 4.6/4.7」 と矛盾する例外、 article_pipeline 内 narration 層は実装側制約優先

### FactChecker strict 度: sentence_type pre-classify pattern
- 旧 prompt = sentence ≒ expected_fact の全 claim 整合判定 → partial coverage と speculative wrapper を mismatch 扱い → 90%+ `regenerate_failed`
- 新 prompt = Python で sentence_type を 'hard_fact' / 'interpretive' に pre-classify、 interpretive は確率 wrapper + partial coverage 許容、 新規捏造数値のみ mismatch
- 実 launch 効果は GitHub Actions cron 21:00 UTC 数日蓄積後に判定

### Starlette `StaticFiles(html=True)` は nested directory の index 自動 serve しない
- `dist/index.html` (root) は OK
- `dist/articles/<slug>/index.html` (nested) は html=True でも 404
- 対処: FastAPI に明示 route `@app.get("/articles/{slug}")` 追加 (5 line bridge、 commit `006f3be`)
- 教訓: 今後 nested SSG ディレクトリを追加する場合は同 bridge pattern が必要、 自動 fallback を期待しない

### Railway build environment は service vars を自動 ARG しない (確証必要)
- Dockerfile に明示 `ARG VITE_SUPABASE_URL` 等を declare する必要あり (commit 既存)
- `SUPABASE_SERVICE_ROLE_KEY` も build-time で必要なら ARG + ENV 必須 (commit `3236794` で追加)
- 但し本 session では結局 anon key + published filter で動いたので、 service_role 経由はまだ実利用していない

### Railway deploy 検知 pattern (bundle hash が local と一致しない)
- Local `npm run build` produces `index-DHBuafZY.js`、 Railway build produces `index-2KNAdu0f.js` (Node ver / npm install state の deterministic 差)
- 故に「local build hash 出現を poll」 pattern は壊れる → endpoint availability 検知 (新 endpoint の 200 / `/articles/<slug>` の 200) を使う
