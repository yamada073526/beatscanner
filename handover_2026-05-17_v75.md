### D. **真っ白事故と修復** (最重要 anchor)

**事象**: StockPriceChart に SMA 統合した直後の deploy で **Pane 2 → ウォッチリスト銘柄選択 → Pane 3 が真っ白**。 backend は完全 healthy だが React mount できず blank。

**原因仮説**: Recharts `<Line dataKey="sma_50" connectNulls>` が **初期 mount 時に全 SMA value = null** (技術的に fetchTechnical 完了前) の状態で render → SVG path で `MNaN,NaN` 生成 → SVG layout 崩壊 → React error → mount 失敗。

**対応**:
1. **緊急 revert**: `git checkout HEAD -- frontend/src/components/StockPriceChart.jsx` で v74 commit 状態に戻す → 真っ白即時復旧
2. **safer 再追加** ([StockPriceChart.jsx](frontend/src/components/StockPriceChart.jsx)):
   - `StockChartErrorBoundary` class 新設 (chart 内部 crash しても Pane 3 親は保護)
   - `default export = ErrorBoundary(StockPriceChartInner)` で wrap
   - SMA fetch は **overlays.length > 0 の時のみ setTechnical** (途中 null 状態を排除)
   - `chartData` は `hasSma50 || hasSma200` の時のみ enrichment、 それ以外は元 `data.prices` (旧挙動と同じ)
   - `<Line dataKey="sma_50">` を `{hasSma50 && (...)}` で conditional render (全 null line を完全に回避)
   - `typeof v50 === 'number'` で SMA value を filter

**memory anchor 候補 (次セッションで作成推奨)**: `feedback_chart_overlay_safety.md`
- ErrorBoundary 必須
- conditional render `{hasFoo && <Line />}` 必須 (全 null line の crash 回避)
- `typeof === 'number' && Number.isFinite` で NaN/Infinity guard
- `isAnimationActive={false}` (Recharts animation 中の null transition crash 既知)

## 3. 触ると危険な箇所 (v75 新規 + 継承)

### v75 新規確定原則

**Recharts `<Line>` 追加時は 3 層防御必須 (handover v75 真っ白事故 SSOT)**:
1. **ErrorBoundary で wrap** (`StockChartErrorBoundary` class、 chart 内部 crash → 親保護)
2. **Conditional render**: `{hasFoo && <Line dataKey="foo" />}` (全 null line を生成しない)
3. **Number guard**: `typeof v === 'number' && Number.isFinite(v)` で NaN/Infinity も弾く
4. **`isAnimationActive={false}`**: Recharts animation 中の null transition crash 既知
- 違反すると Pane 3 全体が真っ白 (React mount 失敗) になる
- handover v75 で `_pick_sample_from_results` 修正と同じく、 「初期 mount 時に全 null state」 を完全排除する設計が必須

**ChartTab.jsx (lightweight-charts) は dead code、 触らない (`feedback_new_ui_only.md`)**:
- 機能追加は **必ず `StockPriceChart.jsx` (Pane 3 Recharts)** に統合
- ChartTab.jsx の SMA 統合 (handover v75 で追加) は dead code として残置 (動作はする)
- Phase 2 lightweight-charts 移行時に salvage 後 delete

**`.backtest-cumchart-*` と `.backtest-equitychart-*` の alias selector (handover v75 CSS rename 漏れ fix)**:
- v74 で旧 → 新 rename したが TradeBarChart JSX は旧 class 残置
- index.css で **両 class を `,` 区切り並列定義** で popover absolute parent 復活
- 将来 TradeBarChart の class を rename する場合は CSS / JSX 両方同時に変更
